## 什么是图形学

图形学，我刚才提到，所谓合成和操作视觉信息，这两样都非常重要，我们之后都会提到。今天，我们着重来说图形学到底有哪些应用，我们为什么要去学习它，它到底好在哪里。

## 图形学在游戏中的应用

举个例子，这有一个《只狼》的游戏。打过的同学都知道，这是一个受苦的游戏，我也非常喜欢。这个游戏本身画面非常好。那么怎么才能辨别在现在这个时代，什么才是好的画面呢？从技术的角度上来说，给大家一个标准——这个标准非常简单：直接看这个画面是不是足够亮就可以了。

为什么呢？这体现了图形学中，或者说渲染中的一个关键技术叫全局光照。如果全局光照做得好，那么整个画面就会亮，看起来就会很舒服。如果你看整个画面比较暗，就会表现出一定程度上的技术不足。从这个角度来说，纯粹从技术的角度，这是一个不错的判定标准。这也是这节课给大家普及的应用知识之一，以后我们会穿插着讲一些这方面的知识。

## 图形学在电影中的应用

在电影里面，图形学得到了广泛的应用。非常早的时候，有一款电影叫做《黑客帝国》，1999 年的电影，已经过去了 20 多年了。即便是今天来看，仍然是一个非常优秀的电影，特别是这些特效，像子弹时间这些东西，就是从这个电影里面衍生出来的。在电影里面叫做 special effects，通过计算机图形学的技术把它合成出来，并且和真实演员、实际存在的东西完美结合在一块，让你觉得看起来非常真实。这就是整个电影特效的工作方式。

特效属于图形学中的一部分。但我多说一句：特效其实是最简单的图形学应用。为什么这么说呢？因为它叫"特效"——你平常见不着它。你平常很难看到这种慢动作的电影，子弹是如何破空、如何形成空气波，透过它可以看到不同折射出来的物体。以及包括出现大爆炸、坏人点燃一个房子、背对着房子走、从来不回头看——你平常见过多少次爆炸？所以说像这些特效，包括爆炸、子弹、一堵墙一般的浪潮涌过来，你平常见得非常少。如果做得不对，你也不会觉得有什么意外。

最困难的东西，往往是日常生活中大家常见的东西。以现在的 rendering 方面的研究为例，大家吃的食物现在做得就非常不好。这是后话了。

## 图形学在动画中的应用

动画这里自然也是不能忽视的。大家应该有很多同学看过 Zootopia（疯狂动物城）。大家都很喜欢毛茸茸的小动物。如果大家仔细看，是真正可以看到一根一根的毛发被做出来、显示出来。不同毛发在一块儿会显示出什么样的效果。想想就觉得非常难——动物身上有多少根毛发呀？百万根够不够？以及每一根头发都要和光线去作用，光线和不同的头发作用，在不同的头发空间弹跳，最后打到人的眼睛里面。我们知道人看到东西，就是因为光线进入了人的眼睛。这么复杂的一个过程，我们应当如何去做？这就是我们之后要说的，首先涉及到几何这块——如何去表述这么复杂的几何形体，就像毛发这种；然后以及如何去渲染，把它真正显示出来，计算光线在这些几何形体之间的传播方式——这个叫做渲染。

最新的动画也是奥斯卡的获奖者——《冰雪奇缘 2》。这里面涉及各种各样不同的技术，比如说 Elsa 放出各种各样的技能，包括烟雾、发光的粒子在空中盘旋，形成不同的图案。这些东西是怎么做出来的？这些东西自然是图形学下面的另外一个重要分支，我们管它叫做模拟或者动画。这块就是在模拟这些微小的粒子如何运动，以及包括更大层面上的，比如说人物衣服在风的吹动下应该如何变化、形状如何变化，头发和头发之间互相有碰撞。把这些东西真实地都模拟出来，我们就可以得到一个真实的样子。这也就是图形学可以研究的东西，在这门课我们也会给大家介绍。

## 图形学在设计与制造中的应用

设计这个事情，大家可能听说过一个缩写叫 CAD——Computer Aided Design。现在的设计很少有手绘设计了，基本上都是用电脑来辅助。举个例子，这里两幅图是同一辆车，只不过左边是电脑渲染出来的（假的，不存在），右边是真实存在的——它在 Autodesk 公司的展厅，我还亲自见到过。它是一辆真实存在的车子，号称不同的部件是 3D 打印出来的，拼成一块。右边是真实的拍摄，旁边有一个电脑，电脑上显示的就是左边这幅图上的东西——这是 CG（computer graphics）生成的车子。

这辆车子允许你做一些交互性的操作，比如你可以移动它，可以换各种各样不同的环境光——在不同环境下，你可以立刻看到它长什么样。这个就非常好。

从这一个例子我们可以看出无数的事情：首先这个车子表面这种光滑的曲线是如何设计出来的——确切来说不是曲线，是曲面，因为它是在三维空间中的。特斯拉最近推出一款非常丑的皮卡，那些面又是如何做出来的？这些都是图形学里面要研究的几何的东西。

## 图形学在家装与室内设计中的应用

大家看到的是一个国外的家装网站——宜家（IKEA）。它这个网站会给大家提供说，比如我们设计出来一套厨房长这样，大家觉得看上去挺好的。可是呢，大家有时候可能意识不到，整个这张图完全是由计算机生成的，而不是一个真的装修结果。对于一些家具和室内的东西，可能只需要艺术家们把不同的部件给摆在一块儿，给一定的光照条件——比如说在这里，大家可以看到光线主要是由右边这个大窗户进来的，进来之后会有各种各样的阴影，光线在这个图形中做各种各样不同的弹跳，最后进入人的眼睛。这个过程叫做渲染（render）。

在宜家的网站上，75% 都是渲染出来的，而这还是几年前的统计，我觉得现在可能会更多。国内现在也有一些类似的公司，比如酷家乐，在做这方面的事情。包括这些室内设计，已经完全走入了平常消费者的范围之内。大家可能平常甚至不太会去关注到，但这些都是图形学在背后的功劳。

## 图形学在可视化中的应用

可视化是我们之前定义图形学的时候提到的——操作视觉信息的方法。可视化就是把一些测量出来的信息变成视觉信息。比如说人体经过三维扫描和 CT、核磁共振等一系列扫描，可以扫描出很多三维空间中的信息，然后通过一定的办法变成视觉信息，这个过程我们就管它叫做可视化。

## 图形学在虚拟现实与增强现实中的应用

虚拟现实（VR，Virtual Reality）和增强现实（AR，Augmented Reality）经常在一块提。它们有什么区别呢？虚拟现实就是你看不到现实——大家戴一个头戴设备，看到的东西全都是电脑生成的，看不到自己面前有一个桌子，互相也看不到对方。增强现实就是说你可以看到现实中的一些东西，并且在这之后还可以看到有一些新的东西叠加进去。

## 图形学在模拟与物理仿真中的应用

模拟是另外一个非常重要的应用，体现在很多方面。比如特效：左边是一个沙尘暴的特效，要模拟各种各样的沙子在空气中怎样飞舞、怎样推过来、怎样把房屋摧枯拉朽。房屋的不同结构会被打成一块一块的，木板中间怎样断裂。包括沙子落到了水里应该长什么样。整个这么一个过程，都需要精确的物理模拟，把过程精确地运算出来——这也是图形学的一个重要部分，叫做模拟。早期有人管这个叫仿真，在美工和 artist 看来这叫动画，都是一回事。

## 图形学在 GUI 与字体设计中的应用

这里提到一个概念叫 GUI——Graphics User Interface，图形用户接口。其实就是不同的设计：有人用过 Mac，有人用过 Windows，有人用过 Linux 各种不同的发行版，各有喜好，觉得哪一种设计比较好。包括 Windows 的磁贴风格、Mac 的各种设计，这些都是属于图形学的范畴，涉及到设计。最近这方面的研究开始独立进行。

还有一个叫做 topography（字体设计）。关于字体方面的设计，大家有没有试过：如果把一张图放大放大再放大，最后会变模糊。但把字体中的字母调字号调到非常大，最大级别，还是会发现它也挺清楚。字体到底是拿什么表示的？和图像有什么区别？为什么在任意放大的情况下都可以看到一个连续的曲线，而图放大就变模糊了？这里就涉及到点阵和矢量这两个不同的概念——之后会给大家解释这个事情。

## 为什么图形学很难

图形学挺难的，为什么？因为它包括了很多东西：你要研究真实世界中各种各样的东西如何作用，包括不同的物质、光、材质——各种各样的东西都需要研究。如果研究不明白，就很难把事情表示清楚。我们需要对真实世界有足够的理解。另外一方面我们要输出——图形学最后要生成一些新东西，这是新的计算方法。然后怎么样把它显示出来？新的显示方法。比如说 Nintendo 3DS 裸眼 3D 技术、全息影像，这些都是新型的显示设备。在不同的显示设备下，这些东西要如何运作？我们提出的方法如何能够用在这些方面上？所以这些东西都加起来，就造成了为什么图形学会相对困难。

但我相信这门课学习了之后，大家会意识到：图形学用到之前各种各样不同的知识，只要咱们把这些前提知识融会贯通、理解清楚，这个事情并不难。

## 课程内容概览

这门课主要包括四个部分的内容：

**第一部分：光栅化（Rasterization）**。把三维空间的几何形体显示在屏幕上，这就是光栅化。现在的 OpenGL 以及所谓 Shader 这些东西是如何运作的，会把这些给大家讲明白。光栅化是所有游戏或者说实时计算机图形学的主要应用。什么叫实时呢？在计算机图形学下面，实时的定义是每秒钟能够生成 30 幅画面，或者叫 30 帧，也就是 30 FPS（frames per second）。能达到这样一个级别，我们就管它叫实时，否则就叫离线（offline）。实时的图形学会广泛应用光栅化的方法，主要是做投影。

**第二部分：曲线和曲面——几何相关的内容。** 如何表示一条光滑的曲线，如何表示曲面，如何用简单的曲面通过细分的方法得到更复杂的曲面，以及当形状发生变化时这些面要如何变化、如何保持物体的拓扑结构。很多这方面内容会给大家介绍，也会讲一些几何方面的前沿。

**第三部分：光线追踪（Ray Tracing）。** 目前动画和电影里面生成真实感画面就是通过光线追踪的方法。因为它的速度慢，但是有好处——它能够生成非常真实的画面。图形学里面有很多事情我们管它叫 trade-off（权衡）——有时候为了达成某一个目标，不得不牺牲一些其他的目标。比如光线追踪能达到好的效果，但就是慢。但人都是贪婪的——正因为这样一个属性，才会让人去寻找两全其美的解决方法。有没有办法让光线追踪做得又像实时光栅化一样快，又能够达到光线追踪的效果呢？当然是有的，现在有新的技术叫做实时光线追踪。在课里面也会给大家介绍。也会讲到一些稍微深入一点的话题，包括路径追踪、更复杂的表面建模以及光线传播方法。

**第四部分：动画与模拟。** 会讲到很多不同的东西：比如扔一个弹性的球，怎么样才能落在地上，如果被挤压如何弹上去再下来。大家可以看到一个例子：把一块布挂好，真实地测出来——左边一列是照片，右边一列是仿真或者模拟，模拟的效果非常令人惊讶。而这已经是算很早以前的技术了，现在只能做得更好。

## 课程不教什么

首先，不说 OpenGL 或者 DirectX。大家打游戏会更多接触 DirectX，Vulkan 算是一个跨平台的图形学 API。这里为什么不教？不是因为这个事情难，而是我希望教大家的是背后的知识。大家学完之后就明白三维到二维如何做投影，如何才把东西真正显示上去。学完了这门课之后，我们就可以运用所学的知识，立刻掌握使用 OpenGL 的方法。

OpenGL、DirectX、Vulkan 这类的东西，我们管它叫做图形学 API。我们用这些东西可以帮我们生成一些矩阵或什么的，但在这个背后，如果我们知道整个工作原理，那么学完这门课之后——我保证——你一定可以自己把 OpenGL 学起来。这就是我们常说的：**We learn graphics, not graphics API。** 我们不想教大家如何使用什么工具。编程语言并不难——我只需要去应用它；而什么东西难呢？背后的算法、数据结构这些东西会困难。道理是一样的。

包括 Shader 怎么样去写——我们教的这套体系里面一定会提及这个事情，只不过不会提及具体的语法，比如 OpenGL 对应的 Shader 语法 GLSL 或者 HLSL 之类。

另外一个不会说的话题是三维建模或者游戏开发。这同样是一些工具：只要我们掌握了如何用 Maya 或者 3ds Max 这类的东西，学习一个星期肯定可以去做一些简单的模型。至于做得好不好，是另外一个事情——处于审美或者美工天赋。但是应用这方面的软件，只要你能够保证图形学的基本概念能够明白，这块绝对不会有问题。

关于使用 Unity 或者 Unreal 这些游戏引擎——它们会包括很多不同的东西：你可以放置一个场景、放置一个人、放置一些灯光、设置一些 AI，就可以正常运作。应用这些东西，我都认为算是相对比较容易的。人们可以用这些工具创建不同的三维模型，设计各种各样的游戏——这些都不会在这个课里面覆盖。

## 图形学与计算机视觉的区别

图形学和计算机视觉往往会被混淆在一块，在这里我希望把这件事情说清楚。在这门课里，咱们不会教计算机视觉。所谓视觉是什么？很简单——一切需要猜测的内容。一切需要猜测的内容基本都是计算机视觉的内容。

比如说左下角看到一张图，我要识别出哪些地方是人、哪些地方是路面、哪些地方是不同的障碍物——我不可能有一个准确的答案，我需要一定程度上的猜测，这就是计算机视觉在做的事情：分析和理解。

深度学习在这门课中不会被特别提及。比如说 GAN 这种结构可以用来生成非常真实的人脸，但也会失败。这一系列方法我们认为是在图像的层面上进行操作，这个操作就不属于图形学的范畴了。涉及到理解、猜测与意义这方面的事情，咱们不会去学习。

我个人有一个理解，可以用一幅图来表达。这幅图最重要的两点：一个叫 model，一个叫 image。Model 更多认为是描述三维空间中的几何形体，或者对于渲染来说是描述材质或者光照这些在三维空间里面一开始就有的东西。然后我们要把它转换成一幅图——这个就叫做计算机图形学，特别是渲染。渲染就是要把这些模型综合在一起考虑，把它变成一幅图——这就是正向的箭头。

反向的箭头：从一张图我如何识别出各种不同的模型？就是我刚才说的语义分割——拍了一张照片，这里面有一个桌子、有一个椅子。从图像我要分析出它到底是一个什么样的结构——这里就是计算机视觉。

另外两条线是一个环形：在 model 这一块自己绕自己（从 model 到 model），这块我认为仍然属于计算机图形学的内容。比如我去描述三维的形体、做三维空间中的仿真——沙子或者油倒到面上——得到的仍然是模型，整个过程中完全没有经历图像。这块我理解的是 model 或者 simulation。

在图像到图像这个圈——从一张图到一张图——以前我们管它叫图像或者视频处理，现在更多的是应用和计算机视觉完全相同的方法，更多是依赖于深度学习。比如从一个人脸变成另外一个人脸——在图形领域上操作，中间涉及到一些猜测和推断推理，这些东西就会更多地运用到计算机视觉的知识。有同学提到三维重建——给你一张图，要把它的三维模型推导出来——这当然是计算机视觉，因为从一张图推它的三维结构，正好就是向左的这一道计算机视觉的连接。

当然，现在的学科之间结合越来越紧密，界限也越来越模糊。有一些问题你可能说不准它到底是属于一个什么样的过程。比如图形学里面我们要应用深度神经网络；AR/VR 的应用中要先用计算机视觉去识别出来一些东西，再放进虚拟场景里面去——又涉及到计算机视觉和图形学的结合。所以这个理解只是给大家做一个参考。

## 作业安排

这次我们的作业是按周来算，每一周有一个小作业。小作业已经提供好了代码框架，大家只要在代码框架上面做一点点改动就可以了。作业有明确的描述：你需要干什么、你需要得到一个什么样的效果都会写清楚。你需要的就是掌握提供的代码框架怎么用，以及快速地把上课教的内容实现进去。

总共有七次小作业。语言要求用 C++。正常情况下代码的工作量很少，每个星期新写的代码应该不超过 20 行。有同学可能觉得 C++ 很头疼，但这里我们提供框架之后，需要用到的知识并不是超级多——特别复杂的指针操作你用不到。我们有非常不错的库可以用，比如说 Eigen，可以做各种方便的矩阵和向量运算。

能不能用 Python 呢？不行。因为 Python 实在是太慢——你单写一个空循环，想生成一幅跟屏幕一般大的分辨率（1280×720），就已经做不到了。那如果你能不做 for 循环、用数组 slice 操作——你既然有这个本事，干嘛不写 C++ 呢？C++ 大家没有任何理由去惧怕它。而且现在几乎所有的工业界都是在用 C++ 来进行图形学方面的编程，所以这块应该是很有用的。

作业框架我们会提供代码框架，同时还会提供一个虚拟机。虚拟机相当于你在任何操作系统环境下，都可以创建一个操作系统中的操作系统——环境是给大家配好的，拿到虚拟机里面立刻就可以用，立刻就可以着手开发。可以省去大家各种编译方面的麻烦。

作业不是一定要写的。如果你要写，就按照这个规矩严格执行，助教会辛苦地批阅。如果你觉得听一听也挺好，我觉得没有任何问题。我们最后试图做一个大作业——从课程差不多中段开始，给大家提供一些想法。大家学了这门课之后可以融会贯通，自己做一套酷炫的东西展示出来。做的最好的一些会被选出来公布在网上。现在我们也正在讨论，做了作业的同学是不是应该得到一些认证。

## 开发环境建议

请大家一定要使用集成开发环境（IDE）。一个好的 IDE 可以自动分析框架里面各种各样不同的东西，在你写代码的时候直接提示：你点一个什么，会有哪些函数可以直接用，语法错误也会直接告诉你——非常方便。

推荐几个 IDE：
- 如果用 Windows：Visual Studio——毫无疑问应该是最好的 IDE。
- Visual Studio Code：跨平台的 IDE，非常好用，在虚拟机里用 VS Code 就非常好。
- Qt Creator（官方念法叫"cute"）：我自己平常用它来分析工程，挺快的。

不推荐的：CLion（我个人不喜欢，感觉影响编程体验），Eclipse（代码分析做得不好，它对 Java 是非常合适的），以及非常不建议用 Sublime Text、Notepad++、Vim、Emacs 这类文本编辑器来做 C++ 工程。这些本身确实都是编辑器，但经过一段时间锻炼之后，特别是以后走向工作岗位，会开发大量庞大的工程——对于这些单薄的文本编辑器来说，功能可能就达不到。能够通过 IDE 增加编程效率，为什么不去用呢？

话说回来，"极客"（geek）这个词是由两个词组合而来的：第一是 genius（天才），第二是 freak（怪胎）。程序员很多这两方面多少都会有一点。我建议大家把这个 geek 中的 genius 这一部分学会，而不要去学 freak 这一块。能把自己的编程水平真正地练上来，通过 IDE 把图形学的工程做好，何乐而不为？
