# 游戏3D场景模型制作规范

> 综合整理自三份项目规范文档，覆盖 PBR 全流程：软件设置 → 模型制作 → UV → 贴图 → LOD → 碰撞 → 引擎导入。

---

## 一、软件版本与基础设置

### 1.1 软件版本

| 软件 | 版本 |
|------|------|
| 3ds Max | 2014 / 2016 / 2017 / 2022 |
| Maya | 2022 |
| ZBrush | 4R7 |
| Substance Painter | 2017 / 2018.1.0 |
| Photoshop | CS6 / 2019 |
| 引擎 | Unity 2018.3.2f1 / UE5 |

### 1.2 Max 基础设置

- **单位**：米（M）
- **坐标**：模型坐标归到原点，位于物件底部；除墙体外的物件，坐标轴正面朝向 **-Y 轴**，Z 轴朝上
- **Reset XForm**：归零后必须执行 Reset XForm
- **光滑组**：以 Max 光滑组为准，禁止使用 Maya 软硬边；光滑组必须与烘焙法线一致
- **模型命名和材质球命名**：必须一致

### 1.3 比例基准

- **以男性角色身高 1.8m 为基准**，合理制作模型大小
- 圆形结构段数：**4 段或 8 段**（低面数优化）
- 室内物件参照标准尺寸（见下文）

### 1.4 Substance Painter 设置

- 项目品质对接 SP 品质
- 新建工程时选择正确的贴图尺寸和格式
- 贴图导出格式为 **TGA**

## 二、模型制作规范

### 2.1 制作流程总览

```
通用流程：
中模 → 高模/卡线模 → 低模 → UV + 烘焙 + UV2 → 贴图 → LOD + 碰撞 → 引擎验证 → 提交

建筑流程：
基础白模 → 贴图规划 + 初贴 → 中模(模块拆分) → 高模/卡线模 → 低模 → UV + 烘焙 + UV2 → 贴图 → LOD + 碰撞 → 引擎验证 → 提交

植物流程：
中模 → 高模 → 低模 → UV + 烘焙 + UV2 → 贴图 → LOD + 碰撞 → 引擎验证 → 提交
（草木灌木类和叶子不需要高模，直接贴图插片）
```

### 2.2 中模规范

- 按人比，根据手机游戏大小，适当删减不必要的细节和结构
- 合并减少细节结构，使物件更加整体
- 树木推荐使用 SpeedTree / ZBrush Z球 快速造型
- 中模阶段要求比例结构合理、剪影丰富

### 2.3 高模规范

| 要点 | 说明 |
|------|------|
| **雕刻造型** | 饱满，不要出现结构穿插的过锐结构 |
| **结构衔接** | 合理自然，不要生硬 |
| **体块节奏** | 明确层次关系 |
| **内凹结构** | 增强切斜角度，保证能烘焙到结构饱满的法线 |
| **倒角原则** | 墙体倒角 > 木头倒角 > 金属倒角 |
| **细节处理** | 重点在外轮廓和体块结构，雕刻大结构/大倒角，勿细碎 |
| **曲面** | 光影正确，没有突兀折痕 |
| **浮片** | 可用浮片制作但需要反倒角，避免过渡生硬 |
| **材质衔接** | 高模制作时应考虑到后续材质衔接 |

> ⚠️ 倒角过小 → 烘焙不出法线或法线结构过小。倒角大小接近写实游戏标准。

### 2.4 低模规范

- **面数分配**：优先外轮廓和大结构，细小结构用 Normal Map 表现
- **布线**：按照结构布线，避免奇怪状的三角面或四边面
- **剪影优先**：尤其是自然资源类
- **概括形体**：非主要的小结构位置全部概括，能烘培在平面上就用平面表示
- **底模不倒角**：倒角尽量烘焙解决
- **UV 和软硬边拆分合理**：打硬边处必须断开，软边看情况连

### 2.5 面数参考

| 物件类型       | 面数范围          |
| ---------- | ------------- |
| 陶瓷罐子（单个）   | 50-200面       |
| 陶瓷罐子（组件）   | 500面以内        |
| 灌木（2米以下）   | 200-300面      |
| 树木（3-6米）   | 1500面以内       |
| 树木（6-12米）  | 2500面以内       |
| 树木（12米以上）  | 3500面以内       |
| 小石头（<2米）   | 100-300面      |
| 中石头（4-8米）  | 300-800面      |
| 大石头（8-12米） | 800-2000面     |
| 大型山石（>12米） | 400-1000面     |
| 超大型山石（>20米） | 400-1000面   |
| 建筑类        | 单mesh不超过7000面 |
| 盆栽植物       | 400-500面      |

> 石头分类归属：<2米归 `detail`（物件类）；4-8米归 `architect`（建筑类）；≥8米归 `structure`（大型物体类）

### 2.6 模型自检清单

- [ ] MAX 单位设置正确
- [ ] 模型尺寸比例正确
- **法线方向**：模型法线方向正确；Normal 贴图格式注意引擎差异——**UE 用 DirectX**（Y-），**Unity 用 OpenGL/Maya 格式**（Y+）。Max 烘焙 Normal 到 Unity 需翻转 Y，到 UE 不需翻转
- [ ] 无未合并点，同一面不超过 4 个顶点
- [ ] 无镂空面 / 重合面
- [ ] 光滑组分配合理（以 Max 为准）
- [ ] 物件坐标归零，位于物体中心底
- [ ] 已执行 Reset XForm
- [ ] FBX 导出正确，清理无用信息（动画/摄像机/灯光）

## 三、UV 规范

### 3.1 UV1（Albedo）

- **最大化使用**贴图UV空间
- 考虑共用和面数，适当放大贴图细节较多或模型占比较大的面
- 背面/底面可适当缩小 UV 面积
- **尽可能填满** UV 空间
- UV **不得超出 0-1 区间**
- **利用率不低于 80%**
- 正面 UV 适当放大，看不到的面尽量缩小
- 旋转公用注意避免过强 Tiling 感

### 3.2 UV2（Lightmap）

- **必须全展**，不能有任何重叠面
- 不能浪费 UV
- UV 断开必须在朝下或内侧部分，**禁止断开朝上或视觉直视点处**
- UV 间距：512 贴图精度上保持 **2-4 像素**距离
- UV 边框间距：保持 **4-6 像素**
- 贴图空间填不满时，**完整空出右侧或下半区域**（方便烘焙时其他模型 UV 自动填充）
- 能合并的 UV 必须合并，接缝隐藏在背面或不显眼处

> LOD0 / LOD1 的 2U 需保持一致；LOD2 不需要 2U

## 四、贴图规范

### 4.1 PBR 流程概述

- 项目使用 **PBR 流程**（金属度/粗糙度工作流）
- 贴图精度：**1米对应 128-512 像素**（依项目而定：手游端游标准不同）
- 单张贴图尺寸上限：**2048×2048**
- 贴图储存格式：**TGA**（24位无Alpha / 32位有Alpha）
- 贴图必须是**正方形**

### 4.2 贴图通道

> ⚠️ 以下为两个项目各自的通道规范，**不可混用**。面试或被问到时需明确区分。

**方案 A — MRA 混合贴图**（Doc2 项目，UE风格）：

| 贴图 | 后缀 | 通道说明 |
|------|:----:|------|
| **Albedo** | `_d` | RGB=颜色；Alpha=透贴（与金属度不同时存在） |
| **Normal** | `_n` | RGB=法线；未单独说明的通道依引擎默认 |
| **MRA** | `_m` | **R=金属度** (0非金属/1金属)；**G=粗糙度** (0光滑/1粗糙，越黑越光滑)；**B=AO** |
| **Emission** | `_e` | RGB=自发光颜色；无Alpha |
| **山石** | — | 只需Albedo+Normal两张贴图 |

**方案 B — Mask 贴图**（Doc1 项目）：

| 贴图 | 后缀 | 通道说明 |
|------|:----:|------|
| **Albedo** | `_d` | RGB=颜色 |
| **Normal** | `_n` | RGB=法线 |
| **Mask** | `_m` | **R=Smooth**（光滑度）；**G=Metallic**（金属度）；**B=AO** |
| **Emission** | `_e` | RGB=自发光颜色 |

> **关键区别**：方案A的R通道是金属度、G是粗糙度；方案B的R通道是光滑度、G是金属度——两套 Mask 的 RG 通道含义完全相反。Normal 贴图方向也因引擎而异（DirectX / Maya 格式）。

### 4.3 PBR 材质参数标准

| 参数 | 标准 |
|------|------|
| **色彩明暗度** | 区间 0.1-0.9，不能过白(1)或过黑(0) |
| **金属度** | 非金属=0，金属=0.8-0.9，不是金属不给金属度 |
| **粗糙度** | 粗糙=1，光滑=0(最光滑不超过0.25)；**越亮越粗糙，越暗越光滑** |
| **灰色裸金属底色** | 明度不低于 0.7 |
| **整体风格** | 亚光为主，不表现很油亮 |
| **贴图明度** | 根据引擎烘焙效果调整，提亮 30%-50% |

### 4.4 贴图制作要点

- 各通道颜色严格按照 PBR 参考表输出，上下不超过 20%
- 忌脏、乱
- **层次结构**：基础底色 → 破损/干裂/风化/褪色 → 污渍/脏渍
- **法线**以大体块为主，减少表面材质颗粒感
- **脏迹和磨损**不过噪、过渡自然，不直接用 SP 遮罩不经处理
- **边缘处理**丰富且不平直
- 贴图共用率高，避免浪费

### 4.5 风格参考

- 参考美术：**绝区零、星穹铁道**
- 风格：**偏写实的风格化处理**（PBR + 手绘效果）
- 区别于纯卡通和纯写实：去除写实中繁杂细碎细节，整体颜色丰富、干净
- 色彩与 Roughness 呈现**彩块化、动漫笔触化**质感
- 室内物件以**干净**为主，颜色保留微弱变化

### 4.6 多贴图规则

- 多张贴图的单个物体，**不能使用多维子材质球**
- 必须一张贴图一个材质球
- 根据模型 ID（贴图）把模型断开

## 五、命名规范

### 5.1 资源分类

| 类别 | 英文标识 | 说明 |
|------|----------|------|
| 建筑 | `build` | 高于6米的建筑类 |
| 树 | `tree` / `bigtree` | 植物类 |
| 石头 | `stone` | 岩石类 |
| 草/灌木 | `grass` | 植被类 |
| 地形 | `terrain` | 地设类 |
| 通用物件 | `small` / `detail` | 6米以下 |
| 大型物体 | `structure` | 32米以上远景 |

### 5.2 命名格式

**建筑类（场景专属）**：
```
地图名_build_编号_变体
例：cemetery_build_01a
    cemetery_build_01b
    cemetery_build_01c
```

**通用物件（跨场景复用）**：
```
Common_物件英文名_编号
例：Common_candle_01
```

**贴图命名**：
```
{模型名}_{后缀}
例：Common_candle_01_d    (Albedo)
    Common_candle_01_n    (Normal)
    Common_candle_01_m    (MRA)
    Common_candle_01_e    (Emission)
```

### 5.3 命名规则

- **全部小写**（LOD 字样例外）
- 不能有空格
- 下划线以外的符号不可使用
- 模型命名和材质球命名需一致
- 通用物件命名前缀加 `common` 标识（部分项目不加，见下方）

> **另一套命名体系**（Doc3 Unity项目）：使用 `地图名_类型_编号_LOD层级` 格式，不加 `Common` 前缀，资源分类为 `architect`(建筑) / `bigtree`(植物) / `detail`(物件) / `structure`(大型物体)。两套体系不可混用。

### 5.4 命名检查清单

- [ ] 模型命名正确
- [ ] Max 材质球命名正确
- [ ] 贴图命名正确
- [ ] Max 文件名命名正确
- [ ] FBX 导出命名正确
- [ ] 截图命名正确

## 六、导出规范

### 6.1 FBX 导出设置

- 导出时选中所有子模型（或建组）导出为**一个 FBX**
- 清理无用模型信息（动画、摄像机、灯光）
- 模型和碰撞**分别导出 FBX**
- 贴图导出格式为 **TGA**

### 6.2 提交文件结构

```
任务名/
├── max/          ← Max源文件
├── fbx/          ← FBX导出文件
├── tex/          ← TGA贴图
├── sp/           ← Substance Painter源文件（可选）
├── zbrush/       ← ZBrush源文件（可选）
└── screenshot/   ← 引擎截图
```

## 七、LOD 规范

### 7.1 层级规则

| 面数范围 | LOD 级数 |
|----------|:--------:|
| ≤500 面 | 不需要 LOD |
| ~1000 面 | 2 级 LOD |
| 1000-3000 面 | 3 级 LOD |
| 3000 面以上 | 4 级 LOD |
| 上万面 (<2.5W) | 5 级 LOD |

### 7.2 减面标准

- **每一级减面 ≥ 50%**
- LOD1：50%
- LOD2：25%
- LOD3：12.5%
- 以此类推

### 7.3 屏幕距离设置

| 级别 | 屏幕距离 |
|:----:|:--------:|
| LOD1 | 0.5 |
| LOD2 | 0.3 |
| LOD3 | 0.2 |
| LOD4 | 0.1 |

### 7.4 移动端特殊规则

- PC 端和手机端模型同步使用
- **手机端模型 = PC 端 LOD1**
- 每个做 LOD 的模型需设置 `Mobile=1`

### 7.5 各类别 LOD 要求

| 类别 | LOD 规则 |
|------|----------|
| **通用物件（<6米）** | 不需要 LOD |
| **建筑类** | LOD0-LOD1-LOD2，每级减 ≥50% |
| **树木（<6米）** | 不需要 LOD |
| **树木（≥6米）** | LOD1 减 ≥50%；LOD2 截屏插片制作（512贴图） |
| **大型山石** | LOD0-LOD1-LOD2 三级 |

> LOD 最后需要**打组导出**，分别命名 LOD0/LOD1/LOD2。LOD 制作推荐使用 Max 自带优化减面工具。

### 7.6 效果验证

- **建筑/树木类**：LOD0 当前视窗 → 间隔 **50米** 放 LOD1 → 再隔 **50米** 放 LOD2
- **大型物体（structure/山石）**：间隔 **100米以上** 查看各级 LOD
- 整体查看效果接近即可
- 提交文件需附带间隔对比效果截图

## 八、碰撞规范

### 8.1 命名格式

```
UCX_物件名_序号
例：UCX_build_01_00
    UCX_build_01_01
```

### 8.2 碰撞制作规则

- 由**简单几何体**制作（Box 或凸起面，不能是凹面）
- 分体模型需**单独命名**
- 尽可能包裹住低模即可，**不需要太多细节**
- 碰撞模型面必须**全封闭**
- **0.5米以上**大小的物件需要制作碰撞（小物件可省略）

### 8.3 碰撞预设

| 物件类型 | 碰撞设置 |
|----------|----------|
| **不可交互**（垃圾桶/电线杆/栏杆/桌子） | 将简单碰撞用作复杂碰撞 |
| **可交互**（楼梯/房子：站姿+攀爬） | 简单与复杂（双层碰撞） |

### 8.4 碰撞行为规则

| 规则 | 说明 |
|------|------|
| 栏杆 | **不可站立** |
| 座椅/桌子 | 可站立，**不可走上跨越** |
| 花坛/花盆/植物 | 只有底座有碰撞，植物无碰撞 |
| 小物件（<家具体积） | **不可站立**，所有小物件都加 |
| 非建筑结构体 | **不可攀爬** |

## 九、引擎导入规范

### 9.1 UE 引擎

- 导入 FBX 时检查材质映射
- LOD 通过命名自动识别（`_LOD0`/`_LOD1`）
- 碰撞通过命名自动识别（`UCX_`前缀）
- 贴图导入后检查压缩格式

### 9.2 Unity 引擎

- Build Settings 设置成 **iOS**
- 使用项目指定 **Shader** 检查效果
- 场景新建平行光（Directional Light）
- 环境设置正确配置
- SP 导出 Unity 贴图格式为 **Unity 5 (Standard Metallic)**
- 贴图阶段就需在引擎中烘焙灯光贴图（2UV），根据烘焙效果调整贴图饱和度/明度

### 9.3 导入检查

- [ ] 模型位置/旋转/缩放正确
- [ ] 材质球数量与贴图对应
- [ ] 碰撞正确加载
- [ ] LOD 正确切换
- [ ] 无漏面/法线反转
- [ ] 引擎内截图提交
