## 二、美术管线详解

### 🔷 2.1 角色美术管线

```
① 概念设计 ────────────────────────────────────────
   ├── 美术风格探索（多方案比较）
   ├── 角色属性定位（战斗/剧情/稀有度）
   ├── 视觉定位（服饰/配色/剪影）
   └── 用户测试验证（海外CE测试）

② 高模制作 ────────────────────────────────────────
   ├── ZBrush 雕刻高模（数百万面）
   ├── Marvelous Designer 布料模拟
   └── Maya/Blender 硬表面建模

③ 低模 & 拓扑 ─────────────────────────────────────
   ├── 手动/自动拓扑（降低面数）
   ├── UV 展开（Texel Density 统一）
   └── 烘焙法线/AO/曲率贴图（Marmoset / Substance）

④ PBR 贴图制作 ────────────────────────────────────
   ├── Substance Painter 绘制（PBR 金属度/粗糙度流程）
   ├── 脸部/皮肤特殊处理（SSS 次表面散射）
   └── LOD 贴图策略

⑤ 绑定 & 动画 ─────────────────────────────────────
   ├── 骨骼绑定 / 蒙皮（Maya/3dsMax）
   ├── Control Rig（UE5 引擎内绑定）
   ├── 动捕数据清洗 & 重定向
   └── 面部动画（MetaHuman / 自定义方案）

⑥ 引擎集成 ────────────────────────────────────────
   ├── 导入 UE5 → 材质实例配置
   ├── 物理资产设置（碰撞/布料/头发物理）
   ├── LOD 生成 & 验证
   └── 性能检查（DrawCall/面数/贴图内存）
```

> **案例：心动《伊瑟》** — 50 人团队，角色管线四阶段并行，节省 1-2 月/角色。概念→设计验证→并行推进（3D细化+动画+特效同步做）→后期（立绘+配音+PV）。

### 🔷 2.2 场景 / 大世界管线

> 📘 深度展开版 → [[场景大世界管线详解]]（含 TA / 美术 / 程序详细分工）

```
① 概念 & 白盒 ──────────────────────────────────────
   ├── 概念设计（参考 80% 真实 + 20% 再设计）
   ├── 白盒搭建（UE5 内几何体快速验证）
   ├── 关卡美术早期介入（灯光/地标初期布局）
   └── 跑测验证（玩法/导航/遮挡）

② 地形制作 ────────────────────────────────────────
   ├── Gaea / WorldCreator / Houdini → 高度图
   ├── UE5 Landscape 导入
   ├── 分层材质（Height/Slope/Curvature 驱动）
   └── Virtual Texture 流式加载

③ 程序化场景 (PCG) ── 🔥 TA 核心战场 ─────────────
   ├── Houdini → HDA 生成器（建筑/植被/道路/河流）
   ├── UE5 PCG Framework（Surface Sampler + Filter + Spawner）
   ├── 植被散布：Poisson Disk / Density Noise / WFC
   ├── 建筑生成：Shape Grammar / 模块化拼装
   └── 性能：HISM 实例化 + LOD 体系 + World Partition

④ 手工打磨 ────────────────────────────────────────
   ├── Hero Asset 手工摆放（视觉焦点区域）
   ├── 灯光布置（Lumen / 传统 Lightmap 混合）
   ├── 后处理（Post Process Volume）
   └── 性能 Profile → 反推 PCG 参数调整

⑤ 场景拆分为：主结构 + 可复用物件
   └── 统一物件风格，减少资产种类
```

### 🔷 2.3 特效管线 (VFX)

```
① Houdini / EmberGen → 模拟（流体/烟雾/破碎）
② Niagara（UE5 粒子系统）→ 实时特效
③ 材质函数 → 特效 Shader（溶解/能量/传送门）
④ Chaos 破坏 → 建筑/场景破坏模拟
⑤ 性能 → Niagara 预算管理、粒子数限制
```

### 🔷 2.4 动画管线

```
① 骨骼绑定 (Rigging)
   ├── Maya/3dsMax 传统绑定
   └── UE5 Control Rig（引擎内绑定，直接调动画）

② 动画制作
   ├── 动捕（光学/惯性）→ 数据清洗
   ├── 手 K 动画（关键帧）
   └── 程序化动画（Motion Warping / IK / Physics）

③ MetaHuman 管线
   ├── Maya建模 → Mesh to MetaHuman
   ├── ZBrush 皱纹细节 → 导出贴图增强
   └── 自定义绑定 + MetaHuman 绑定合并
```

## 三、TA 在管线中的介入点

| 阶段 | TA 工作 | 产出 |

## 四、UE5 核心技术栈与大厂选型

| 技术 | 用途 | 大厂策略 |
|------|------|----------|
| **Nanite** | 高模直接渲染，免 LOD | PC/主机项目标配；移动端暂无 |
| **Lumen** | 动态全局光照 | PC/主机标配；移动端用 Lightmap |
| **World Partition** | 大世界流式加载 | 开放世界项目必用 |
| **PCG Framework** | 程序化场景生成 | 大世界标配；与 Houdini 互补 |
| **Niagara** | 粒子/VFX | 统一特效系统 |
| **Chaos** | 物理/破坏 | 3A 级破坏效果 |
| **MetaHuman** | 高精度角色 | 叙事/过场动画项目 |
| **Houdini Engine** | HDA 在 UE 内 Cook | PCG 管线核心 |
| **Virtual Texture** | 大世界贴图流送 | 减少显存占用 |

## 五、PCG TA 专项 — 程序化场景管线

### 🔶 5.1 Houdini ↔ UE5 数据流

```
                    ┌─────────────┐
                    │   Houdini   │
                    │  (离线生成)  │
                    └──────┬──────┘
          ┌────────────────┼────────────────┐
          ↓                ↓                 ↓
    ┌──────────┐    ┌──────────┐     ┌──────────────┐
    │ Heightmap│    │  FBX/USD │     │ Point Cloud  │
    │  (RAW)   │    │ (静态资产)│     │ (CSV/JSON)   │
    └────┬─────┘    └────┬─────┘     └──────┬───────┘
         ↓               ↓                   ↓
    ┌──────────┐    ┌──────────┐     ┌──────────────┐
    │Landscape │    │Static Mesh│     │ PCG Data     │
    │ Import   │    │ Import   │      │ Table/Asset  │
    └──────────┘    └──────────┘     └──────────────┘
                                            ↓
                                    ┌──────────────┐
                                    │ PCG Graph    │
                                    │ (UE5 实时生成)│
                                    └──────┬───────┘
                                           ↓
                                    ┌──────────────┐
                                    │ HISM Spawner │
                                    │ (实例化渲染)  │
                                    └──────────────┘
```

### 🔶 5.2 混合工作流四大原则

| 原则 | 说明 |
|------|------|
| **关注点分离** | Houdini = 离线复杂算法/模拟；UE5 PCG = 实时实例化+LOD；蓝图 = 逻辑粘合 |
| **数据流单向** | HDA 制作 → 导出中间格式 → PCG 读取 → 图表处理 → 蓝图控制 |
| **属性继承** | Houdini 属性（`instance_id`/`variant_id`/`scale_var`/颜色）→ PCG Point 属性 |
| **确定性保证** | 统一 Seed 种子系统，离线+实时可复现 |

### 🔶 5.3 工业化 PCG 团队分工

```
策划/关卡 → 白盒搭建
                ↓
Houdini TA → 程序化生成工具(HDA)
                ↓
地编/美术 → UE 内 PCG Graph 调参 + 材质变体 + 微调
                ↓
TA → 性能 Profile → 反推 PCG 参数优化
                ↓
程序 → 蓝图 API + 运行时 PCG 集成 + 性能底层
```

### 🔶 5.4 开源方案：Undini Pipeline Framework

- **功能**：一键 UE Spline → Houdini 批处理 → CSV/FBX 回导 → 自动创建 PCG Graph
- **协作设计**：美术只在 UE 内点按钮；TA 配置 HIP 和参数
- 支持 UE 5.3.2+ / Houdini 20+

## 六、大厂面试高频流程题

> **Q: 描述一个完整的 UE5 游戏项目美术管线？**
>
> 概念设计 → DCC建模(ZBrush/Maya/Blender) → 贴图(Substance) → 绑定动画 → 导入UE5 → 材质/Shader → 灯光/后处理 → 性能优化 → 打包
>
> TA 在每个环节的介入点和产出。

> **Q: UE5 PCG 和 Houdini 各自负责什么？**
>
> Houdini：复杂算法（侵蚀模拟/WFC/地形生成）、离线批处理、HDA 封装
> UE5 PCG：实时散布（植被/石头）、LOD 管理、引擎内快速迭代
> 两者互补，不是替代关系。

> **Q: 你用 UE5 做过什么 PCG 相关的项目？**
>
> 用 STAR 方法，讲清楚技术方案选择 + 遇到的坑 + 如何解决 + 最终效果。

> **Q: World Partition 和传统 Level Streaming 的区别？**
>
> World Partition = 自动网格划分 + 按需加载；传统 Streaming = 手动切关 + 手动触发。WP 更适合大世界协作开发（OFPA 一人一文件）。
