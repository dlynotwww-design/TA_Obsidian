## 二、TA（技术美术）的具体工作

> TA 不是某个人的岗位，而是**贯穿所有环节的角色**。在大世界管线中，TA 是连接 Houdini、UE5 PCG、美术和程序的桥梁。

### 各阶段 TA 介入详情

#### Phase B：白盒阶段

| 工作内容 | 具体产出 |
|---------|---------|
| 提供 UE5 内白盒工具模板 | 预置几何体库、快速测量工具 |
| 跑测数据收集脚本 | 自动记录玩家路径、视线范围、面数分布热力图 |
| World Partition 初始划分方案 | Cell Size / Loading Range 配置建议 |

#### Phase C：地形阶段

| 工作内容 | 具体产出 |
|---------|---------|
| Gaea/Houdini → UE5 地形导入流水线 | 高度图 / 权重图导入脚本 + 参数预设 |
| **自动分层材质系统（母材质）** | `MF_LandscapeAuto` — 根据 Height/Slope/Curvature/Erosion/HoleMask 自动混合 8+ 层材质 |
| Virtual Texture 配置 | RVT Volume 设置、Page Table 参数调优 |
| 地形材质性能优化 | 减少 Texture Sample、合并 Layer、使用 Material Layer Blend |

#### Phase D：PCG 阶段 🔥 — TA 最核心战场

| 工作内容 | 具体产出 |
|---------|---------|
| **Houdini HDA 工具开发** | 建筑布局生成器 / 道路系统 / 河流工具 / 植被散布器 |
| **PCG Graph 设计 & 封装** | 通用 PCG 模板（森林 / 草地 / 城市 / 废墟）供地编使用 |
| Houdini → UE5 数据桥梁 | 属性映射规范（`point_scale` → PCG Scale，`variant_id` → Mesh Variant） |
| **种子系统（Seed System）** | 确保离线 Houdini + 实时 PCG 使用统一 Seed 可复现 |
| HISM 实例化策略 | 合并阈值、LOD 切换距离、Culling Distance 自动设置 |
| 程序化规则编写 | Poisson Disk 散布 / 坡度过滤 / 遮挡剔除 / 生态关联 |

#### Phase E：手工打磨阶段

| 工作内容 | 具体产出 |
|---------|---------|
| Shader & 材质库维护 | 母材质库（标准 PBR / 风格化 / 植被 Wind / 水面 / 半透明） |
| 灯光预设 & 规范 | 不同 TOD（Time of Day）的灯光参数标准 |
| 后处理 LUT & 材质 | 风格化调色 LUT、自定义后处理材质（风格化 / 描边 / 像素化） |
| 工具脚本 | 批量替换资产 / 材质实例自动生成 / 命名检查 |

#### Phase F：性能优化阶段

| 工作内容 | 具体产出 |
|---------|---------|
| RenderDoc 帧分析 | DrawCall 瓶颈定位、Overdraw 热力图、贴图内存审计 |
| Unreal Insights 分析 | Game Thread / Render Thread 瓶颈定位 |
| 性能预算表制定 | 每个区域的 DrawCall / 面数 / 贴图内存 / 灯光数上限 |
| **反推 PCG 参数** | "这块区域 DrawCall 超了 30% → 提高 HISM 合并阈值 → 重生成" |
| 自动检查工具 | CI/CD 中集成资产合规检查（面数 / 贴图尺寸 / LOD 数量） |
| LOD 审核 | 确保所有资产有合理 LOD 链、Screen Size 切换距离正确 |

### TA 核心产出总结

```
TA 的"武器库"：
├── Houdini HDA 库（建筑 / 植被 / 道路 / 河流 / 城市）
├── PCG Graph 模板库（森林 / 草地 / 废墟 / 矿区）
├── 母材质库（地形 / 植被 / 建筑 / 特效 / 水 / VFX）
├── 自动化工具（导入 / 检查 / 报告 / 批量操作）
├── 性能规范文档（面数 / 贴图 / DrawCall / 灯光预算）
└── 灯光 / 后处理标准（TOD 预设 / LUT / PP Volume 模板）
```

## 三、美术的具体工作

### 概念美术

- 根据世界观设定产出**情绪版（Mood Board）**和**关键帧（Key Art）**
- 确定区域配色方案、光影基调、视觉风格方向
- 收集真实世界参考（**80% 真实 + 20% 再设计** 原则）
- 绘制区域规划图：标注生态分布、建筑风格、光照氛围

### 3D 场景美术

| 工作内容 | 工具 | 说明 |
|---------|------|------|
| 模块化资产制作 | Maya / Blender / 3dsMax | 墙体、地板、屋顶、门窗等可复用模块 |
| 高模雕刻 | ZBrush | 岩石、雕像、遗迹等特殊资产 |
| PBR 贴图绘制 | Substance Painter / Designer | 金属度 / 粗糙度流程贴图 |
| 遵循 TA 母材质规范 | UE5 Material Instance | 使用母材质创建实例，而非新建材质 |
| LOD 制作 | 引擎内 / Simplygon | 或交给自动工具 + TA 审核 |
| Texel Density 统一 | — | 通常 512px/m 或 1024px/m |

### 植被美术

- **SpeedTree** 制作树木 / 灌木骨架和叶片
- 制作草 / 花 / 石头 / 蘑菇等小资产
- 配置 Wind Material（按 TA 提供的植被母材质创建实例）
- 制作 LOD（通常 3-4 级：高模 → 减面 → Impostor / Billboard）

### 关卡美术 / 地编（Level Artist）

> 地编是大世界管线的**核心美术角色**，负责把一切拼成最终画面。

| 工作内容 | 说明 |
|---------|------|
| **白盒搭建** | 在 UE5 内用 BSP / 基础模型快速验证空间关系 |
| **地形雕刻** | 在 Landscape 上手雕关键区域（悬崖、山谷、河床） |
| **PCG 调参** | 使用 TA 提供的 PCG Graph 模板，调整密度 / 变体 / 排除区域 |
| **Hero Asset 摆放** | 在视觉焦点区域手工放置高精度资产 |
| **灯光布置** | 主光方向、补光、氛围光、点光源点缀 |
| **后处理调色** | 调整 Exposure、Color Grading、Bloom 等参数 |
| **场景叙事** | 通过资产摆放和灯光讲"环境故事" |
| **跑测验证** | 反复在关卡中跑图，调整视觉节奏和引导 |

### 场景拆分策略

```
场景拆分为：主结构 + 可复用物件
│
├── 主结构（手工搭建）
│   ├── 地形（Landscape）
│   ├── 大型建筑 / 地标
│   ├── 道路 / 桥梁
│   └── 特殊地形（洞穴 / 悬崖）
│
└── 可复用物件（PCG 散布）
    ├── 植被（树 / 草 / 灌木）
    ├── 石头 / 碎石
    ├── 小建筑（房屋 / 棚屋）
    └── 装饰物（围栏 / 路灯 / 招牌）
```

> 统一物件风格，减少资产种类，提高复用率。

## 四、程序的具体工作

### 引擎 / 渲染程序

| 工作内容 | 说明 |
|---------|------|
| **World Partition 配置** | Cell Size、Loading Range、HLOD 层级、Streaming Source 设置 |
| **Lumen 调优** | 间接光照质量 / 反弹次数 / 更新速度 / 回退方案 |
| **Nanite 配置** | 哪些资产启用 / 禁用 Nanite、Fallback Mesh 设置 |
| **Virtual Shadow Maps** | 阴影分辨率、Cache 策略、移动端回退方案 |
| **RVT (Runtime Virtual Texture)** | 地形 + 建筑贴花混合的底层实现 |
| **Shader 开发** | 特殊 Shader（风格化 / 特效 / 后处理），TA 提需求 → 程序实现底层 |

### Gameplay 程序

| 工作内容 | 说明 |
|---------|------|
| **流式加载系统** | 基于玩家位置的异步加载 / 卸载 Cell |
| **LOD 系统** | Cull Distance Volume / LOD 切换逻辑 / Screen Size 计算 |
| **HISM / ISM 管理** | 实例化渲染的 Culling / LOD / 动态开关 |
| **物理 & 破坏** | Chaos 破坏系统的触发条件与性能管理 |
| **交互系统** | 采集 / 攀爬 / 游泳 / 滑翔等与场景的交互逻辑 |
| **导航网格（NavMesh）** | AI 寻路所需的地面覆盖 |

### 管线 / Tools 程序

| 工作内容 | 说明 |
|---------|------|
| **资产导入 Pipeline** | FBX/USD 批量导入、自动材质分配、命名检查 |
| **CI/CD 资产检查** | 自动检查面数 / 贴图尺寸 / LOD 数量 / 命名规范 |
| **PCG 运行时集成** | 蓝图 API 暴露 PCG 参数给玩法系统 |
| **性能 Profiling 工具链** | Unreal Insights 配置 / 自动化性能测试 |
| **多平台适配** | PC / 主机 / 移动端的渲染路径切换、质量档位 |
| **数据序列化** | 大世界关卡数据的存储格式与版本管理 |

## 五、TA - 美术 - 程序 协作关系图

```
策划 / 关卡
    │  提出需求："这里需要一个森林，那里需要一个村庄"
    ↓
  ┌─ TA ─────────────────────────────────────────────┐
  │  "我来做生成工具"                                  │
  │  → Houdini HDA：森林生成器 / 村庄生成器             │
  │  → PCG Graph：封装好参数，地编调参即可              │
  │  → 性能预算：这片森林最多 50W 面 / 200 DrawCall     │
  └────────────────────┬─────────────────────────────┘
                       ↓
  ┌─ 美术（地编）─────────────────────────────────────┐
  │  "我来让它好看"                                    │
  │  → 调 PCG 参数：密度、变体、排除区域                │
  │  → 手工摆放 Hero Asset：大树、地标建筑、篝火        │
  │  → 布置灯光、调后处理                               │
  │  → 跑测验证视觉节奏                                 │
  └────────────────────┬─────────────────────────────┘
                       ↓
  ┌─ 程序 ───────────────────────────────────────────┐
  │  "我来保证能跑"                                    │
  │  → 运行时 Streaming 不出 Bug                       │
  │  → LOD 切换无感                                    │
  │  → 60fps 稳定（RenderDoc → 优化 DrawCall/Shader）  │
  │  → 物理 / 碰撞 / 导航网格正确                       │
  └────────────────────┬─────────────────────────────┘
                       ↓
  ┌─ TA（再次介入）───────────────────────────────────┐
  │  "我来做性能闭环"                                   │
  │  → RenderDoc 抓帧 → 发现 DrawCall 超预算            │
  │  → 反推 → PCG HISM 合并阈值调整 → 重生成            │

  │  → 更新性能规范文档                                 │
  └──────────────────────────────────────────────────┘
```

> **核心循环**：TA 提供工具 → 美术使用 → 程序保障 → TA 优化 → 反推参数 → 迭代

### 混合工作流四大原则

| 原则 | 说明 |
|------|------|
| **关注点分离** | Houdini = 离线复杂算法 / 模拟；UE5 PCG = 实时实例化 + LOD；蓝图 = 逻辑粘合 |
| **数据流单向** | HDA 制作 → 导出中间格式 → PCG 读取 → 图表处理 → 蓝图控制 |
| **属性继承** | Houdini 属性（`instance_id` / `variant_id` / `scale_var` / 颜色）→ PCG Point 属性 |
| **确定性保证** | 统一 Seed 种子系统，离线 + 实时可复现 |
