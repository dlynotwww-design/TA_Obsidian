## 一、PCG 核心能力（简历最强项，必深挖）

### Q1: 你在"弥华宇宙"项目里用纯 UE 蓝图开发了室内 PCG 功能，具体实现了哪些节点？和 UE 原生 PCG 节点比，你扩展了哪些？

**简历对应**：「纯UE蓝图开发，自定义节点扩展PCG功能」

**面试官想验证**：是否真的懂 PCG 底层，还是只会拖官方节点？

**准备要点**：
- 讲清楚自定义了哪个节点（继承 `UPCGSettings` / `UPCGNode`），做了什么原生节点做不到的事
- 举例：室内空间的"房间分割→家具匹配→碰撞避让"这一链路的蓝图逻辑
- 如果用了蓝图 PCG 自定义节点，要说清楚性能考量（蓝图 vs C++ 节点）

### Q2: 600m×600m 概念城市是怎么做的？"手动制作街区单元模块，结合 PCG 关卡实例散布"——这里的关卡实例（Level Instance）和普通 Static Mesh Spawner 有什么区别？为什么选关卡实例？

**简历对应**：「600m*600m网格概念城市……手动制作街区单元模块，结合PCG关卡实例散布」

**面试官想验证**：对 UE5 Level Instance 的理解深度，以及技术选型的判断力。

**准备要点**：
- Level Instance = 可包含完整 Actor 层级的预制关卡片段（含灯光/碰撞/蓝图），远超 Static Mesh Spawner 的能力
- 街区单元用 Level Instance → 每个街区是"完整小关卡"，可以独立编辑、独立 Cook
- 但 Level Instance 有性能开销，600×600 用了多少个实例？如何管理的？
- 和 Packed Level Actor 的取舍？

## 二、Houdini 能力

### Q9: "Houdini 解算资产导出 VAT，在 UE 中二次编辑"——VAT 的原理是什么？导出和导入的完整流程？

**简历对应**：「Houdini解算资产导出VAT，在UE中二次编辑」

**面试官想验证**：对 VAT（Vertex Animation Texture）技术的真正理解。

**准备要点**：
- VAT = 把顶点动画烘焙到贴图中（Position Texture + Normal Texture），运行时在 Shader 中还原
- Houdini 侧：SideFX Labs VAT 工具 → 导出两张 EXR 贴图
- UE 侧：VAT Shader（读取贴图 → 偏移顶点位置）
- **优势**：一个 DrawCall 播放复杂动画（群集、破碎），比骨骼动画便宜几十倍
- 你在 UE 中二次编辑了什么？(贴图调整? 动画速度? 混合?)

## 九、面试官可能挖的坑

### ⚠️ 坑1：「你的 Grasshopper 城市工具和 UE PCG 城市工具，哪一个更厉害？」

**陷阱**：不要贬低任何一个。它们定位不同——GH 是参数化方案探索，UE PCG 是游戏资产输出。两者互补。

### ⚠️ 坑2：「你说开发效率提升 60%+，怎么算的？」

**陷阱**：不要编数据。准备一个具体的计算逻辑：原来做 X 要 N 小时，现在 M 小时，(N-M)/N = 60%。

### ⚠️ 坑3：「你一个人做了这么多东西（PCG+Shader+特效+工具），时间怎么分配的？深度够吗？」

**陷阱**：这是检验你是否"广而不精"。挑一个最深入的方向重点展开，其他方向坦率说是"能上手但非专家"，并表示 PCG 是你的核心深度。

### ⚠️ 坑4：「你们团队几个人？你说联动 UI 和程序，UI 和程序各几个人？」

**陷阱**：验证项目真实性。小团队不是缺点，但要说清楚分工和自己的角色。万维猫的项目显然更正式，弥华宇宙可能是小团队/独立项目——坦率说清楚。

### ⚠️ 坑5：「建筑学转 TA，你觉得自己比纯计算机/纯美术出身的 TA 差在哪里？」

**陷阱**：不要说"不差"，要承认差异但突出互补。比如：C++ 能力可能不如纯计算机出身，但空间理解和美术审美强于纯程序员；美术功底可能不如纯美术出身，但逻辑和算法能力强于纯美术。
