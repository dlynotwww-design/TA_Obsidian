## 根据资料，使用了PCG（程序化内容生成）技术的主要项目及其细节整理如下：

### 1. **重建秦朝古都 (腾讯天美 T1 / 效能产品部)**

这是一个旨在通过程序化方法高效构建中国古代大都市的研究项目。

- **技术细节**：
    - **三阶段工具集**：1. **手动布局**，由美术放置代表区域和道路的立方体/曲线；2. **交互式生成**，在区域内生成建筑物“子盒子”，美术可实时调整；3. **细节生成**，在庭院内填充高精资源和装饰，达到交付质量。
    - **城市布局算法**：基于秦朝“**闾里制**”规则，在Houdini中开发了递归划分算法；利用**动态规划 (DP)**寻找输入区域的**最大内接矩形**，确保“英雄建筑”（如宫殿）的放置空间。
    - **架构与管线**：采用**Houdini云服务后端**进行布局计算，通过**USD (通用场景描述)**标准实现跨工具（UE与Houdini）数据交换。
    - **特定工具**：开发了**三维锚点系统**（控制建筑拉伸时不破坏视觉特征）、**共线编辑模式**（基于几何算法的交互微调）以及多种POI工具（如城墙自动适配地形、稻田生成器）。
- **成效**：将城市构建成本从预计的 **336个人月降低至9个人月**。

### 2. **三角洲行动 (Delta Force - 腾讯天美 JADE)**

该项目针对100平方公里的大规模开放世界，利用PCG解决了多端（PC与移动端）性能瓶颈。

- **技术细节**：
    - **生态系统生成**：通过**Houdini离线工作流**抓取地形数据，根据规则生成点云并返回引擎实例化，驱动包括冻土、沙漠在内的多种写实生态。
    - **Clipmap 缓存技术**：将超大地图的环境信息（植被状态、水体数据、地形湿度等）压缩进一张环形缓存贴图中，将显存开销从 **100MB 压降至 2.5MB**。
    - **多端差异化优化**：针对低配移动端，通过PCG生成特定拓扑的道路模型以适配降维地形，并利用**随机最大权重ID (Stochastic max weight ID)** 算法优化悬崖渲染，在极低采样下消除接缝。

### 3. **Genesis Scene (腾讯 AI 框架)**

这是一个基于意图驱动的场景编辑框架，展示了PCG与AI的深度融合。

- **技术细节**：
    - **联结主义 + 符号主义**：利用大语言模型（LLM）理解用户的自然语言或草图意图（联结主义），转化为参数后由确定性的**符号化PCG引擎**执行生成。
    - **全栈生成**：包含基于气候模拟（温度/降水）的生物群落分配、基于**建筑文法 (Building Grammars)** 的城市布局，以及具有语义逻辑的室内家具摆放（如卧室内自动放床）。
- **成效**：实现 **10平方公里全场景在10秒内生成**，地形再生仅需3秒。

### 4. **风暴之城 (Against the Storm - Eremite Games)**

这款Roguelite城市建造游戏将PCG与玩法核心机制进行了深度集成。

- **技术细节**：
    - **动态平衡网络**：地图上的林间空地、资源和遗迹完全过程化生成。
    - **非线性负反馈生成**：系统在生成新挑战或蓝图选项时，会动态评估当前玩家的物资缺口和居民满意度，确保生成的随机内容在策略上是“可解”的。

### 5. **无限暖暖 (Infinity Nikki - 叠纸游戏)**

在打造高品质跨平台开放世界中应用了PCG相关的优化技术。

- **技术细节**：
    - **花海优化**：针对大面积花海，将花卉资源烘焙到**SVT (流式虚拟纹理)** 中，解决远距离剔除后的视觉缺失问题，在不损失性能的前提下保证远景美观。
    - **程序化动画**：结合UE5原生功能开发了**自研程序化动画工具**，实现精细的布料模拟和实时穿插处理。

### 6. **太平洋驱动 (Pacific Drive - Ironwood Studios)**

该项目展示了小团队如何利用PCG工具链实现大规模关卡产出。

- **技术细节**：
    - 利用 **Houdini Engine** 开发了数字资产（HDA），支持美术在UE编辑器内将粗糙的地形草图动态细化，自动提取斜率特征并生成景观材质层权重图。

### 7. **其他应用PCG算法的项目**

资料还提到了多项基于特定PCG算法的项目：

- **Townscaper、Bad North、Caves of Qud**：广泛应用了**波函数坍缩 (WFC)** 算法及其变体来处理局部拼缝和结构合规性。
- **矩阵觉醒 (The Matrix Awakens)**：应用了 WFC 算法进行大规模城市生成。
- **原神 (Genshin Impact)**：在其 UGC 模块“**摩拉克斯乐园**”中，允许玩家利用分层柏林噪声 (Perlin noise) 生成随机地形。
- **Embark Studios (Arc Raiders, The Finals)**：利用 Houdini 的 PDG (过程化依赖图表) 构建柔性管线，为多款 Live 运营游戏提供程序化变体资产。

# 近十年（2016-2026年）是 PCG（Procedural Content Generation，过程生成技术）从“小众独立游戏的随机生成”**走向**“3A大作工业化管线核心”的黄金十年。

在 GDC、SIGGRAPH、CEDEC 等顶级技术大会上，PCG 的应用演变可以清晰地划分为三个阶段：**美术资产工具化（Houdini 生态）、大世界生态模拟、以及当下的运行时实时生成与虚幻5（UE5）PCG 框架**。

以下是近十年中，上述大会关于 PCG 技术应用最具有里程碑意义的硬核会议与演讲整理：

### 一、 3A 开放世界工业化管线（2016 - 2019）

这一时期 PCG 的核心诉求是“效率”。大厂开始利用流程化工具（如 Houdini）来生成海量地形、道路和植被，把美术师从重复劳动中解放出来。

- **《地平线：零之曙光》中的全自动化世界构建 (GDC 2017)**
    
    - **分享者：** Guerrilla Games
        
    - **技术亮点：** 详细拆解了如何用基于规则的 PCG 算法，在一夜之间生成整个游戏的生态系统（植被拓扑、岩石分布、河流侵蚀）。美术师只需绘制高维度的“密度控制图”，算法就会自动处理层叠关系和生物群落边界，极大提升了 3A 开放世界的开发效率。
        
- **《幽灵行动：荒野》的地形工具与技术 (GDC 2017)**
    
    - **分享者：** Ubisoft (育碧巴黎)
        
    - **技术亮点：** 这是 **Houdini Pipeline** 在 3A 工业界的最典型应用。育碧展示了如何用 PCG 技术生成 400 平方公里的复杂玻利维亚地形。不仅包括了山脉和河流的物理侵蚀模拟，更绝的是“村落/建筑生成器”——根据地形斜率、道路可达性自动规划房屋位置，并自动保证建筑的门窗、结构符合美术规格。
        

### 二、 纯数学进化与玩法机制的深度融合（2018 - 2021）

在这一阶段，PCG 不仅用于生成“好看的背景”，更开始深入到游戏的关卡流派（Roguelike）和底层算法优化中。

- **游戏程序员的数学：程序化配方的威力 (GDC 2018)**
    
    - **分享者：** Squirrel Eiserloh
        
    - **技术亮点：** 探讨了 PCG 在数学层面的进阶方案。提出“离线生成海量内容并进行自动化筛选（Vetting），最终只存储精简配方（Recipe Input）”的思路，彻底解决了传统 PCG 饱受诟病的“生成内容过于随机、缺乏设计感、容易穿模”等通病。
        
- **《死亡细胞》：离散网格中的程序化关卡设计 (GDC 2019)**
    
    - **分享者：** Motion Twin
        
    - **技术亮点：** 典型的“自顶向下 (Top-Down)”的关卡 PCG 经典案例。分享了如何用图论（Graph Grammars）先生成关卡的“主干与分支”逻辑线（哪里是钥匙、哪里是锁、哪里是 Boss），然后再将手工艺人制作的“房间房间块（Room Chunks）”有机地拼接进去，实现了“既有精妙的局部微操设计，又有无限重玩性”的完美平衡。
        

### 三、 实时物理模拟与高精度环境生成（SIGGRAPH 核心，2020 - 2023）

学术界的 PCG 更倾向于物理真实性，这些论文直接反哺了现代游戏引擎的物理模拟。

- **大规模地形与流体侵蚀的实时耦合模拟 (SIGGRAPH 2021/2022)**
    
    - **发表者：** 各种顶尖高校与 NVIDIA / Unity 实验室
        
    - **技术亮点：** 传统的地形 PCG 只是用柏林噪声（Perlin Noise）垫底，而近几年的学术突破在于将 **水力侵蚀（Hydraulic Erosion）** 和 **热力风化（Thermal Weathering）** 写进 GPU 计算。游戏内的地形可以根据虚拟降雨量，实时冲刷出河道、淤泥堆积和山体滑坡，让自然景观的生成达到了以假乱真的地步。
        

### 四、 工匠精神：细节渲染与二次元 PCG（CEDEC 核心，2022 - 2024）

日本 CEDEC 大会的 PCG 往往透露出极致的“实用主义”和对特定美术风格的偏执。

- **《怪物猎人：崛起 / 荒野》的生态与植被程序化交互 (CEDEC 2022/2023)**
    
    - **分享者：** CAPCOM (卡普空)
        
    - **技术亮点：** 分享了 RE 引擎如何利用 PCG 在复杂地形上快速生成符合生物学规律的生态圈，并且探讨了 PCG 生成的物理草丛、灌木如何与大型怪物的高频碰撞进行**实时、低消耗的响应**。
        
- **日系二次元大作中的自动化“破损与污渍”生成 (CEDEC 2023)**
    
    - **分享者：** Square Enix 等
        
    - **技术亮点：** 针对非写实渲染（NPR / 动漫风）游戏，如何用 PCG 算法自动在 3D 角色的衣服折角、建筑边缘生成符合二次元审美的“卡通边缘线、脏迹与破损效果”，避免了美术师手动去画贴图，保证了风格的高度统一。
        

### 五、 新纪元：运行时实时 PCG 与引擎内嵌生态（2023 - 2026 现状）

近三年 PCG 迎来了爆发，最大的催化剂是 **Unreal Engine 5 的 PCG 框架** 走向成熟。PCG 从“美术编辑器的外挂”变成了“游戏运行时（Runtime）也可以实时计算的底层”。

- **《Electric Dreams》与虚幻引擎 5 的全新 PCG 框架体系 (GDC 2023 / Unreal Fest)**
    
    - **分享者：** Epic Games 官方
        
    - **技术亮点：** 这是 PCG 历史上的重大分水岭。Epic 展示了引擎内置的实时 PCG 图表（PCG Graph）。它允许开发者在不借助 Houdini 的情况下，在 UE5 内部直接用节点编写高阶的生成逻辑，并与 Nanite 和 Lumen 完美契合。**“运行时 PCG”** 的出现，意味着游戏甚至可以根据玩家的实时行为、剧情走向，在几毫秒内实时动态生成、修改方圆几公里的森林或城市。
        
- **《黑神话：悟空》的大规模中式古建筑与自然环境生成管线 (GDC 2025 / 业界交流)**
    
    - **技术背景：** 随着中式魔幻、写实大作的崛起，如何将海量的实景扫描（Photogrammetry）资产转化为游戏内合理的排布成为了新课题。在近期的技术分享中，如何利用 PCG 算法将扫描得到的复杂古建筑组件（如斗拱、瓦片、石阶）根据空间算法和历史规制进行自动化“组装”和群落散布，是 3A 工业化落地的新范式。
        

### 总结：近十年 PCG 的技术进化轨迹

| **演进维度** | **10年前（2016左右）**      | **5年前（2021左右）**   | **2025-2026 现状**              |
| -------- | --------------------- | ----------------- | ----------------------------- |
| **主要工具** | 独立引擎底层代码 / 基础 Houdini | 高阶 Houdini / 引擎插件 | **UE5 内置 PCG** / 实时 GPU 节点    |
| **生成时机** | 100% 离线生成（烘焙进游戏包）     | 离线生成为主 + 部分运行时拼砖  | **完全运行时（Runtime）实时计算生成**      |
| **美术控制** | 比较死板，容易穿模或千篇一律        | 美术绘制 Mask，控制力大幅提升 | 美术与程序员在同一张**逻辑图表**上协同         |
| **核心算法** | 噪声函数 / 细胞自动机 / 汪氏砖    | 物理侵蚀模拟 / 语法文法树    | 空间采样（Spatial Sampling） / 语义理解 |
