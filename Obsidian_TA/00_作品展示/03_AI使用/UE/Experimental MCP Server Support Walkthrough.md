[Experimental MCP Server Support Walkthrough](https://www.youtube.com/watch?v=AlV__BFg8qk)

这段视频详细展示了 **Unreal Engine 5.8** 中引入的**实验性 MCP（模型上下文协议）服务器支持**，旨在通过大语言模型（LLM）降低游戏开发的门槛，减少技术开销，并提高创作效率。

以下是视频内容的详细概括：

### 1. MCP 服务器的核心理念
*   **连接桥梁：** MCP 服务器充当了 Unreal Engine 与各种模型（如 Claude Code、Gemini 或自定义模型）之间的桥梁。
*   **保持控制权：** 虽然模型可以执行操作，但开发者依然保留对场景的完全编辑能力，这与生成无法修改的静态图像完全不同。
*   **开放性：** 该系统是开放的，开发者不被绑定在特定模型或工作流上。

### 2. 游戏开发工作流的变革
*   **自然语言构建场景：** 通过在终端输入指令（如“现代客厅”），模型会利用 **5.8 版本中的语义搜索**功能从资产库中提取并摆放家具。
*   **大规模城市生成：** 开发者可以指示模型生成公路网、划分地块并填充森林。系统生成的不是简单的渲染图，而是**真实的几何体和可编辑的 PCG（过程化内容生成）图表**。
*   **多人协作：** 支持多用户会话，不同开发者构建的资产放入城市时，道路和地块会自动调整以适应变化。

### 3. 专业技能与逻辑嵌入（Skills）
*   **灯光技能：** 开发者只需描述想要的效果（例如“波哥大上午 9:30 的阳光”），模型会自动调节大气、天空、后期处理等数十个复杂参数。
*   **材质与特效逻辑：** 模型可以**阅读现有的材质节点图**，根据指令编写新逻辑并插入正确位置。此外，它还能扩展 Niagara 粒子系统模板，通过单一参数控制多个发射器。
*   **蓝图与 UMG：** 开发者可以通过提示词构建 HUD 界面（如危险计），并由蓝图逻辑将触发器、材质和特效系统串联起来。

### 4. 线性内容创作与 AI 引导（针对影视制作）
*   **深度信息引导（Conditioning）：** 使用 **Movie Render Graph** 将 UE 场景的深度 Pass 传递给图像模型（如 Nanobanana Pro），使生成的 AI 图像在构图和透视上与引擎场景完全吻合。
*   **3D 资产重建：** 视频展示了如何从 AI 生成的图像中提取特定物体（如轮胎），通过 AI 模型（Triple 3.1）将其转化为**带纹理的 3D 网格体**并导入 UE 中。
*   **视频生成与风格化：** 利用“风格帧”和深度 Pass，通过视频模型（如 Seed Dance 2.0）生成具有相机控制权的视频。
*   **实时扩散（Diffusion）预览：** 演示了将 UE 视口实时流传输到局部扩散模型，实现快速的风格化尝试，如将场景瞬间转变为木炭画或水彩画风格。

### 5. 总结与展望
视频强调，MCP 服务器和相关工具（如 PCG 原始插件、技能库）已在 5.8 版本中发布，旨在**降低技术摩擦**，让原本需要数月的工作在几天内完成。官方表示，未来将继续致力于在降低复杂性的同时，确保艺术家拥有对创作的**完整控制权**。
