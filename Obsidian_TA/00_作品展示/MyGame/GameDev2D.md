## 二、引擎选型

### 2.1 Godot + AI

| 方面 | 说明 | 优势 |
|------|------|------|
| **核心引擎** | Godot 4.x，使用 GDScript 或 C# 开发 | 开源、轻量、2D 渲染成熟 |
| **AI 辅助** | 通过插件或 API 接入大语言模型（LLM） | 用于生成对话、任务文本、地图命名等 |
| **程序化生成** | 使用 Godot 内置的 Noise 类（FastNoiseLite） | 柏林噪声、WFC 算法原生支持 |
| **社区资源** | Godot 官方 2D 示例项目丰富 | 快速上手，减少造轮子 |

**集成方案**：
- 使用 `HTTPRequest` 节点调用本地或云端 LLM API
- 通过 `GDScript` 解析 AI 返回的 JSON，动态生成游戏内容
- 例如：AI 生成 NPC 对话 + 任务描述，引擎生成对应 NPC 并放置在场景中

### 2.2 TaptapMaker

| 方面 | 说明 | 适用场景 |
|------|------|----------|
| **定位** | 轻量级 2D 游戏制作平台 | 快速原型验证、小型独立游戏 |
| **优势** | 无需编程、模板丰富、一键发布 TapTap | 策划/美术快速测试想法 |
| **限制** | 扩展性有限，不支持复杂逻辑 | 不适合大型或复杂项目 |

**建议**：前期可使用 TaptapMaker 快速验证玩法原型，确定方向后迁移至 Godot 进行正式开发。

## 五、程序实现

### 5.1 地形算法

#### 5.1.1 WFC（波函数坍缩）

**核心思想**：从局部模式推断全局布局，确保相邻元素之间的约束一致性。

**在 2D 游戏中的应用**：

| 应用场景 | 输入 | 输出 |
|----------|------|------|
| **地图生成** | 小块地图模板（如草地、森林、建筑） | 完整地图瓦片网格 |
| **室内布局** | 房间模块（客厅、厨房、走廊） | 完整房间组合 |
| **建筑立面** | 窗户、门、墙砖的局部模式 | 完整建筑外观 |
| **装饰铺设** | 少量地面装饰模式 | 丰富的地面细节 |

**Godot 实现方式**：
- 使用 `GridMap` 或 `TileMapLayer` 作为底层数据结构
- 实现 WFC 核心算法（观察、坍缩、传播、回溯）
- 可通过 GDScript 或 C# 编写，性能敏感部分建议用 C#

#### 5.1.2 柏林噪声

**核心思想**：通过连续噪声函数生成自然的渐变值，模拟自然地形。

**在 2D 游戏中的应用**：

| 应用场景 | 使用方法 | 效果 |
|----------|----------|------|
| **地形高度图** | 2D 噪声值映射为地形类型 | 草地 → 森林 → 山脉的渐变过渡 |
| **生物群落分布** | 多噪声图叠加（温度+湿度） | 不同区域呈现不同生态 |
| **树木/资源密度** | 噪声值高于阈值时放置 | 自然的资源分布 |
| **云层/水波** | 随时间采样的动态噪声 | 流动的视觉效果 |

**Godot 实现方式**：
```gdscript
# Godot 4.x 示例
var noise = FastNoiseLite.new()
noise.seed = randi()
noise.frequency = 0.05

for x in range(width):
    for y in range(height):
        var value = noise.get_noise_2d(x, y)
        # 将值映射为地形类型
        if value < -0.3:
            set_cell(x, y, TERRAIN.WATER)
        elif value < 0.1:
            set_cell(x, y, TERRAIN.GRASS)
        elif value < 0.5:
            set_cell(x, y, TERRAIN.FOREST)
        else:
            set_cell(x, y, TERRAIN.MOUNTAIN)
```

### 5.4 渲染管线

**Godot 2D 渲染核心机制**：

| 要素 | 说明 | 优化建议 |
|------|------|----------|
| **CanvasItem** | 所有 2D 节点的基类 | 使用 `is_visible_in_tree()` 控制可见性 |
| **TileMap** | 地图瓦片绘制 | 使用 Atlas 纹理，合并减少绘制调用 |
| **Light2D** | 2D 动态光照 | 限制光照范围，使用 Shadow 缓存 |
| **Shader（着色器）** | 自定义视觉效果 | 使用 `CanvasItem` 着色器实现水体、天气等效果 |
| **视口分层** | 前景/背景独立渲染 | 使用多个 `SubViewport` 实现视差效果 |

**渲染顺序管理**：
```
背景层 (Background) → 地形层 (Terrain) → 装饰层 (Decoration)
→ 对象层 (Objects) → 角色层 (Characters) → 前景层 (Foreground) → UI 层
```
使用 `CanvasItem.z_index` 控制层级，`YSort` 节点实现垂直排序（类等距视角）。

**性能优化要点**：
- 使用 **纹理图集（Texture Atlas）** 合并小纹理，减少绘制调用
- 利用 **可见性剔除**，只渲染屏幕范围内的 TileMap 内容
- 对远处资源使用较低的更新频率
- 使用 `MultiMeshInstance2D` 渲染大量相同物体（如树木、草）
