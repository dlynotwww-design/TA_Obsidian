# MyGame 「癫狂的世界」— 开发规划

## Context

用户（建筑学出身 + AIGC全栈）想做一个**受 Messenger 启发的浏览器端 3D 探索游戏**，作为个人创作主线。游戏核心理念来自库哈斯《癫狂的纽约》：用建筑表达情绪，创造一个「癫狂」的城市空间。

**MVP 范围**：单场景步行模拟——一座「垂直分裂塔」，每层风格迥异，WASD移动+鼠标视角，纯客户端。

**技术路线**：Three.js + Vite + TypeScript，用 Claude AI 辅助编程迭代。

**项目位置**：`D:\TA\MyGame\`（Vault 外部独立目录）

## 阶段一：项目脚手架与基础运行（Day 1）

### 目标
搭建最小可运行的 Three.js + Vite + TypeScript 项目，能在浏览器中看到一个 3D 场景。

### 步骤
1. 在 `D:\TA\MyGame\` 创建 Vite + vanilla-ts 项目
2. 安装 Three.js 及必要依赖
3. 配置 Vite（tree-shaking、路径别名）
4. 创建基础场景文件结构：
   ```
   D:\TA\MyGame\
   ├── index.html
   ├── package.json
   ├── tsconfig.json
   ├── vite.config.ts
   ├── public/
   └── src/
       ├── main.ts            # 入口：初始化 Engine + GameLoop
       ├── core/
       │   ├── Engine.ts      # 单例：THREE.Scene、Camera、Renderer、Clock
       │   ├── GameLoop.ts    # requestAnimationFrame 循环、delta 计算
       │   └── InputManager.ts # Keyboard + Mouse 输入封装
       ├── scenes/
       │   └── MainScene.ts   # 当前 MVP 唯一场景
       ├── entities/
       │   └── Player.ts      # 玩家角色控制器
       ├── utils/
       │   └── constants.ts
       └── style.css
   ```
5. 实现一个带光照、地面、基本立方体的场景
6. 确认 `npm run dev` 能在浏览器中运行

### 关键参考

## 阶段二：玩家角色控制器（Day 1-2）

### 目标
实现 WASD 移动 + 鼠标拖拽旋转视角 + 第三人称跟随相机。

### 步骤
1. **InputManager** 封装键盘输入（W/A/S/D 持续按下状态、空格跳跃）和鼠标输入（按下拖拽旋转、滚轮缩放）
2. **Player 实体**：
   - 一个代表玩家的胶囊体/立方体
   - 速度、加速度、摩擦力
   - `update(delta)` 方法：根据输入计算移动方向，更新位置
3. **Third-person Camera**：
   - 相机在玩家后方固定偏移
   - 鼠标拖拽旋转相机围绕玩家
   - 滚轮调整距离
   - Spring/damping 平滑跟随
4. 添加地面碰撞检测（简单的地面高度检测即可，MVP 不需要完整物理引擎）

### 架构决策
- MVP 阶段**不使用物理引擎**（Rapier/cannon-es），用简单的 AABB 碰撞 + 地面高度检测
- 输入系统用「轮询模式」而非「事件模式」——在每帧 update 中检查按键状态

## 阶段三：球形小世界（Day 2-3）

### 目标
参照 Messenger，将世界放在一个小型球面上，角色在球面上行走，重力指向球心。

### 步骤
1. **球面世界**：创建一个半径约 50 单位的球体 Mesh（半透明/线框模式用于调试）
2. **球形重力系统**：
   - 每帧计算球心到玩家位置的方向向量
   - 重力方向 = `-directionToCenter.normalized()`
   - 玩家始终被「吸附」在球面上（表面 + 固定高度偏移）
   - `camera.up = directionToCenter.normalized()`（确保相机始终以球心为「上」）
3. **球面移动**：
   - WASD 方向映射到球面的切平面
   - 玩家始终「站立」在球面上
4. 在球面上放置一些测试建筑块（立方体/圆柱体）

### 关键参考
- Messenger 的核心技术：`camera.up = playerPosition.normalized()`，重力指向球心
- 重力系数：`g = (planetRadius / distance)^2` 或简化为常量指向球心

## 阶段四：建筑区域设计——垂直分裂塔（Day 3-5，核心创作阶段）

### 目标
在球形世界表面放置**多个建筑集群**，每个集群代表一种「癫狂」的风格。MVP 的核心是**一座垂直分裂塔** + **2-3 个其他风格区域**。

### 建筑区域规划

#### A. 垂直分裂塔（中央主建筑）
库哈斯「1909定理」——每层是独立世界：
```
层数  主题                        建筑风格                        情绪
────  ──────────────────────────  ────────────────────────────  ────
顶层   悬浮禅园（古典中式园林）     飞檐 + 假山 + 雾气             宁静/超脱
↑     宋风书院（木构 + 书卷）      层叠斗拱 + 纸窗                沉思
│     赛博市场（霓虹 + 密集摊位）   玻璃幕墙 + 全息广告            躁动
│     苏联公寓（粗野主义混凝土）    预制板 + 裸露管道              压抑
│     哥特教堂（尖拱 + 彩色玻璃）   飞扶壁 + 玫瑰窗                敬畏
│     蒸汽机房（齿轮 + 管道迷宫）   铆接钢板 + 蒸汽喷口            焦虑
底层   下沉废墟（被掩埋的古典柱式） 残柱 + 泥土 + 根系             遗忘
```

#### B. 漂浮碎片区（Antigravity Precipice）
参考 Beomsik Won 的反重力系列——建筑碎片悬浮在球面一侧的空中。

#### C. 拼贴街区（Collage District）
参考 Matthias Jung——正常住宅建筑与不可能元素的拼接（一扇窗有房子那么大、屋顶漂浮分离）。

#### D. 拥挤谷（Congestion Valley）
参考库哈斯「拥挤文化」——建筑极度密集、缝隙狭窄、视觉压迫。

### 资产生成策略
- **AI 生成纹理**：用 SD/MJ 生成每种建筑风格的贴图（砖墙、木纹、玻璃、霓虹等），导入项目
- **程序化几何体**：建筑主体用 Three.js 内置几何体（BoxGeometry、CylinderGeometry 等）组合搭建
- **GLB 模型（可选）**：如果有 Blender 建模能力，导出少量精细模型作为 Hero Asset
- **颜色/材质区分**：不同区域用不同色调（温暖→寒冷/宁静→躁动），增强情绪表达

### 文件结构扩展
```
src/
├── world/
│   ├── Planet.ts           # 球面世界管理
│   ├── GravitySystem.ts    # 球形重力 + 表面吸附
│   └── zones/
│       ├── TowerZone.ts    # 垂直分裂塔
│       ├── FloatZone.ts    # 漂浮碎片区
│       ├── CollageZone.ts  # 拼贴街区
│       └── ValleyZone.ts   # 拥挤谷
├── assets/
│   └── textures/           # AI 生成的贴图
```

## 阶段五：氛围与交互增强（Day 5-6）

### 目标
让场景「有感觉」——光照、后处理、音效、简单的交互反馈。

### 步骤
1. **动态光照**：
   - 主方向光模拟太阳（可缓慢旋转模拟昼夜？MVP 固定日间即可）
   - 每个建筑区域有独特的点光源/聚光灯（霓虹粉、暖黄、冷蓝等）
   - 环境光保持基础可见度
2. **雾效**：远处的建筑在雾中若隐若现，增强距离感和神秘感
3. **后处理**（可选，视性能而定）：
   - 轻微的 Bloom 效果（霓虹区域发光）
   - 轻微的色差/Vignette（增强「癫狂」感）
4. **音效**（Web Audio API 程序化生成，不依赖音频文件）：
   - 脚步声（不同地面材质不同音调）
   - 区域环境音（风、机械嗡嗡声、远处钟声）
   - 进入新区域的过渡音效
5. **交互反馈**：
   - 靠近建筑时显示区域名称（HTML overlay 或 CSS 浮标）
   - 简单的粒子效果（漂浮灰尘、萤火虫）
6. **性能优化**：
   - 远离玩家的区域降低渲染精度或隐藏
   - InstancedMesh 用于重复元素（窗户、柱子等）

## 阶段六：阅读体验层——库哈斯式叙事（Day 6-7）

### 目标
不只是「看建筑」，而是通过**文字片段**让玩家理解这个世界的「理论」。模仿《癫狂的纽约》中库哈斯为不存在的建筑撰写宣言的方式。

### 叙事实现
1. **区域铭牌系统**：
   - 玩家走到特定位置触发文字浮现（HTML overlay，淡入淡出）
   - 文字以「建筑理论宣言」的口吻写就，模仿库哈斯风格
   - 示例：
     > *「此塔于1909年被发现——或发明。它的每一层都不知晓相邻层的存在。这是一种垂直的分裂，一种空间的脑白质切除术。」*
2. **碎片收集**（可选，小而美的收集系统）：
   - 每个区域散落 2-3 个「理论碎片」——发光的文本卡片
   - 走近自动收集，显示在屏幕角落
   - 收集足够多后解锁关于这个世界的「真相」

### 叙事文本主题
- 每一段文字都是对库哈斯某个概念的「游戏化转译」
- 曼哈顿主义 → 「这座塔是拥挤的庆典，而非问题」
- 脑白质切除术 → 「立面与内部早已离婚，各自再婚」
- 偏执批判法 → 「你看到的一切都是一个论点的伪造证据」
- 乐观现实主义 → 「不要修复混乱，学会在混乱中冲浪」

## 技术栈总览

| 层 | 选择 | 理由 |
|----|------|------|
| **3D 引擎** | Three.js r175+ | WebGL 标准，社区成熟 |
| **构建** | Vite + TypeScript | 快速 HMR、tree-shaking、类型安全 |
| **物理** | 无（自定义碰撞） | MVP 不需要完整物理引擎 |
| **后处理** | Three.js EffectComposer（可选） | 轻量 Bloom/Vignette |
| **音效** | Web Audio API | 程序化生成，零文件体积 |
| **资产** | 程序化几何体 + AI生成纹理 | 快速迭代，小体积 |
| **部署** | Vercel / Netlify | 免费、自动 HTTPS、CDN |

每个阶段完成后：
1. `npm run dev` → 浏览器打开 → 确认场景正常运行
2. `npm run build` → 检查 dist/ 大小
3. 录屏/截图 → 放入 Obsidian 开发日志
4. 提交 Git → 可回滚

最终验证：
- 浏览器打开线上 URL → 完整走一遍所有建筑区域
- 移动端测试（可选，Messenger 支持手机）
- 发给 2-3 个朋友试玩，收集反馈
