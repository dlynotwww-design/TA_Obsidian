# 游戏开发中美术与程序岗位职责及工作内容产出

> 游戏开发是高度协作的工业流程，美术负责"玩家看到什么"，程序负责"游戏如何运行"。技术美术（TA）是两者之间的关键桥梁。

---

## 一、美术岗位（Art）

### 1. 概念美术（Concept Artist）

**职责**：
- 根据策划需求创作视觉概念图
- 确定项目美术风格方向
- 为 3D 美术提供设计参考

**产出物**：
- 角色概念设定图
- 场景气氛图（Key Art）
- 道具/载具设计稿
- 色彩脚本（Color Script）
- 故事板/分镜（Storyboard）

---

### 2. 3D 场景美术（Environment Artist）

**职责**：
- 制作游戏场景、建筑、地形、植被等 3D 资产
- 搭建关卡白盒与最终场景
- 优化场景性能与视觉表现

**产出物**：
- 场景模型（.fbx/.usd/.obj）
- 材质与贴图（PBR 流程：BaseColor/Normal/Roughness/Metallic/AO）
- 地编关卡文件
- LOD 模型
- 场景布局与构图

**常用工具**：Maya、Blender、3ds Max、ZBrush、Substance Painter/Designer、Houdini、UE/Material Editor

---

### 3. 3D 角色美术（Character Artist）

**职责**：
- 制作角色、怪物、NPC 的 3D 模型
- 雕刻高模并烘焙法线贴图
- 配合绑定师完成模型拓扑

**产出物**：
- 高模/低模角色
- PBR 贴图
- 表情 Blendshape（可选）
- 角色展示渲染图

---

### 4. 动画师（Animator）

**职责**：
- 为角色、怪物、载具制作动画
- 设计动画状态机与 blend tree
- 与策划、程序配合实现 gameplay 动画

**产出物**：
- 骨骼动画（.fbx/.anim）
- 动画片段（Idle/Walk/Run/Attack/Death 等）
- 面部动画
- 过场动画（Cinematic）
- 动画状态机配置

---

### 5. 特效美术（VFX Artist）

**职责**：
- 制作技能特效、环境特效、UI 特效、爆炸/火焰/烟雾等
- 使用粒子系统、Shader、序列帧等技术
- 平衡特效视觉表现与性能

**产出物**：
- 粒子特效（Niagara/Particle System/VFX Graph）
- Shader/材质特效
- 序列帧贴图与 Mesh 特效
- 特效模板与复用资产

---

### 6. 灯光美术（Lighting Artist）

**职责**：
- 负责场景灯光氛围与视觉叙事
- 调整曝光、色彩、阴影、全局光照
- 与渲染、后处理配合实现最终画面

**产出物**：
- 场景灯光方案
- Lightmap/Lightprobe 设置
- 后处理体积（Post Process Volume）配置
- 日夜/天气氛围方案

---

### 7. UI/UX 美术（UI Artist / UX Designer）

**职责**：
- 设计游戏界面、图标、HUD
- 制定 UI 视觉规范与交互逻辑
- 确保信息层级清晰、操作流畅

**产出物**：
- UI 界面设计稿（Figma/Sketch/PSD）
- 图标、按钮、血条、地图等 UI 资源
- UI 动效设计
- 交互原型与规范文档

---

### 8. 技术美术（Technical Artist）

**职责**：
- 搭建美术管线与工具链
- 编写 Shader、脚本、PCG 工具
- 解决美术与程序之间的技术问题
- 性能优化与资产规范制定

**产出物**：
- Shader/材质库
- Python/蓝图/PCG 工具
- 美术规范文档
- 自动化脚本（批量导入/检查/导出）
- 性能优化方案

**细分方向**：
- Shader TA
- 工具/管线 TA
- PCG TA
- VFX TA
- 动画/绑定 TA

---

## 二、程序岗位（Programming）

### 1. 游戏玩法程序（Gameplay Programmer）

**职责**：
- 实现游戏核心玩法逻辑
- 与策划、美术配合落地战斗、移动、交互系统
- 编写可维护的游戏系统代码

**产出物**：
- 玩家控制器、战斗系统、任务系统代码
- Gameplay 框架与工具接口
- 策划配置表读取逻辑
- 玩法 Demo 与原型

---

### 2. 客户端/引擎程序（Engine Programmer）

**职责**：
- 维护与扩展自研引擎或商业引擎
- 优化渲染、内存、加载等底层性能
- 实现跨平台支持（PC/主机/移动端）

**产出物**：
- 渲染管线代码
- 资源加载与内存管理模块
- 平台适配层
- 性能剖析与优化报告

---

### 3. 图形程序（Graphics Programmer）

**职责**：
- 实现与优化渲染特性
- 开发 Shader、后处理、光照方案
- 与 TA 协作落地高级视觉效果

**产出物**：
- 渲染特性代码（光照、阴影、雾、水体、体积云等）
- HLSL/GLSL/Shader Graph 底层 Shader
- 后处理效果
- 渲染性能优化方案

---

### 4. 工具程序（Tools Programmer）

**职责**：
- 开发编辑器插件与内部工具
- 提升美术、策划、测试的工作效率
- 维护资产管线与自动化流程

**产出物**：
- DCC 插件（Maya/Blender/Houdini/Max 插件）
- 编辑器扩展
- 自动化构建/打包工具
- 数据校验工具

---

### 5. AI 程序（AI Programmer）

**职责**：
- 实现 NPC 行为、寻路、决策系统
- 设计敌人 AI、群集行为
- 与策划配合调整 AI 参数

**产出物**：
- 行为树/状态机/GOAP 系统
- 寻路与导航网格代码
- AI 配置文件
- 战斗/巡逻/交互逻辑

---

### 6. 网络程序（Network Programmer）

**职责**：
- 实现多人联机架构
- 处理同步、延迟、反作弊
- 优化网络性能

**产出物**：
- 客户端-服务器通信模块
- 状态同步与预测逻辑
- 网络协议设计
- 反作弊与安全模块

---

### 7. UI 程序（UI Programmer）

**职责**：
- 实现 UI 界面逻辑
- 与 UI 美术配合实现动态效果
- 处理输入、事件、数据绑定

**产出物**：
- UI 框架代码
- 界面控制器与数据绑定
- UI 动画与过渡效果
- 多分辨率适配方案

---

### 8. 音频程序（Audio Programmer）

**职责**：
- 实现音频播放、混音、3D 音效
- 集成 Wwise/FMOD 等中间件
- 优化音频性能

**产出物**：
- 音频系统代码
- 音效触发逻辑
- 动态音乐系统
- 音频资源管理模块

---

## 三、美术与程序的协作流程

```text
策划需求
   ↓
概念美术出设计稿
   ↓
3D 美术制作资产
   ↓
TA 制定规范、写 Shader/工具
   ↓
程序实现玩法与渲染支持
   ↓
整合到引擎关卡中
   ↓
测试、迭代、优化
```

### 典型协作点

| 环节 | 美术负责 | 程序负责 | TA 负责 |
|------|----------|----------|---------|
| 角色制作 | 模型、贴图、动画 | 动画系统、状态机 | 绑定规范、Shader、工具 |
| 场景搭建 | 资产、布局、灯光 | 加载系统、LOD、流送 | PCG 工具、性能优化 |
| 特效 | VFX 资产 | 粒子系统接口 | Shader、VAT、特效管线 |
| UI | 界面设计稿 | UI 框架与逻辑 | UI Shader、动效工具 |
| 性能优化 | 按规范制作资产 | 渲染与加载优化 | 制定规范、检查工具 |

## 四、技术美术（TA）为什么是桥梁

TA 是少数既懂美术语言又懂程序语言的岗位：

- **对美术**：把复杂的技术需求翻译成可执行的制作规范
- **对程序**：把美术需求整理成清晰的技术方案
- **对项目**：建立管线、工具、标准，让美术资产能高效、正确地进入引擎

### TA 的典型工作场景

1. 美术想做一种特殊材质效果 → TA 写 Shader 或配合图形程序实现
2. 大世界场景手工摆放太慢 → TA 开发 PCG 工具
3. 资产导入引擎总是报错 → TA 写自动化检查脚本
4. 游戏帧率不够 → TA 分析 Draw Call、材质、LOD，制定优化方案
5. 美术和程序对规范理解不一致 → TA 制定并维护美术/技术规范文档

---

## 五、总结

| 岗位大类 | 核心关注点 | 典型产出 |
|----------|------------|----------|
| **美术** | 玩家看到的内容 | 模型、贴图、动画、特效、UI、灯光 |
| **程序** | 游戏运行的逻辑与性能 | 系统代码、渲染特性、工具、网络、AI |
| **TA** | 美术与程序之间的效率与质量 | Shader、工具、管线、规范、优化方案 |

在大规模项目中，**美术决定视觉上限，程序决定技术上限，TA 决定落地效率**。三者的高效协作是游戏品质的关键。
