## 目标

实现两个空间（A场景与B场景）之间的无缝切换效果。

玩家触发切换后，在角色前方生成一个空间球体，球体逐渐扩张。球体范围内显示目标空间，范围外保留当前空间，形成两个空间同时存在的视觉效果。

当玩家进入球体范围后，完成空间穿越，并卸载当前场景，仅保留目标场景。

系统支持 A→B 与 B→A 双向切换，并保证切换过程中玩家状态、游戏逻辑及运行流程不中断。

---

## 总体思路

采用 Level Streaming、Material Parameter Collection（MPC）、Sphere Mask、Dissolve 以及 Niagara 等技术实现空间切换效果。

切换时不直接进行关卡跳转，而是在切换开始时提前加载目标场景，通过 Sphere Mask 控制两个场景的显示区域，实现两个空间同时存在的视觉表现。

Sphere Mask 范围外显示当前场景，Sphere Mask 范围内显示目标场景。随着空间球体不断扩张，目标空间逐渐侵蚀当前空间。

当玩家进入空间球体后，系统判定完成穿越，卸载原场景，仅保留目标场景，从而完成空间切换。

### 流程示意

```text
触发切换
    ↓
加载目标场景
    ↓
生成空间球
    ↓
Sphere Mask扩张
    ↓
球内显示目标空间
    ↓
玩家进入球体
    ↓
卸载原场景
    ↓
完成切换
```

## 关卡流

系统采用 Level Streaming 方式管理场景，实现多个空间的动态加载与卸载。

### 关卡结构

```text
Persistent Level
│
├─ Player
├─ BP_TransitionManager
├─ Level_A
└─ Level_B
```

### Persistent Level

负责：

- Player
    
- GameMode
    
- UI
    
- BP_TransitionManager
    

不参与空间切换。

### Level_A

当前空间。

### Level_B

目标空间。

## 主要材质

### 场景物体主材质控制显示范围

所有参与空间切换的场景物体统一继承：

```text
M_Master_Transition
```

aipha控制默认是否显示：

通过 MPC 获取球形遮罩需要的相关数据以及材质A的状态：

- DissolveHardnessA
    
- DissolveRadiusA
    
- DissolveInvertA
    
- DissolvePositionA
	
- ActivateA

切换过程中：

```text
当前空间：
球外显示
球内隐藏

目标空间：
球内显示
球外隐藏
```

最终形成：

```text
球外 = 当前空间

球内 = 目标空间
```

### 溶解效果

为了增强空间侵蚀感，在 Sphere Mask 基础上叠加 Noise 纹理，通过噪声扰动打破规则球体边界

材质逻辑：

```text
SphereMask × Noise × NoiseStrength
```

### 发光边缘

```text
MaskA = SphereMask(R)

MaskB = SphereMask(R + Width)

Edge = MaskB - MaskA
```

输出至：

```text
Emissive
```

## 蓝图流程

### 状态变量

- Where A：当前处于 A 空间（true）还是 B 空间（false）
- PortalOpened?：传送门是否已完全打开
- LevelsValid：A、B 关卡是否都有效
- Activate / ShowA / ShowB：Timeline 驱动的过渡状态
- Radius：当前球体半径（Timeline Lerp 驱动）
- Investigationmode?：调查模式（可选扩展）

### 主要事件流程

#### 触发阶段（Overlap Event）

- 玩家进入 Portal Sphere → ComponentBeginOverlap
- 根据 Where A 判断切换方向
- 调用 PortalAOpen 或 PortalBOpen

#### PortalAOpen / PortalBOpen（以 A→B 为例）

1. LoadStreamLevel（目标关卡，异步）
2. 设置 Streaming Level 属性，bShouldBeVisible
3. CollisionEnableB_Pre：
    通过 Tag for B 批量启用目标空间物体（可见性 + 碰撞）
4. DistortionFX + Timeline 驱动球体扩张（Radius 从 0 → PortalMaxRadius）
5. Niagara 粒子同步缩放
6. OffsetPortalOpen：玩家位置轻微偏移（防止卡在边界）
7. 玩家进入球体 → UnloadStreamLevel（原空间）
8. CollisionEnableB 

### MPC控制SphereMask位置及缩放

游戏运行中，Tab键触发球体生成事件，获取玩家位置，根据玩家位置及预设的球体半径计算球体位置，写入 MPC
### Timeline控制遮罩缩放：

时间线设置3秒，同步更新：

- Sphere Collision
    
- Niagara Radius

## Niagara

### 匹配球体尺寸，生成环绕粒子，粒子跟随球体一起缩放

主要组成：环绕粒子，卷曲噪波力

粒子生成模式：

```text
Sphere Locaton
```

暴露参数：

```text
- User.MeshScale
- User.ParticleColor1 / Color2
- User.ParticleFade
- User.SpawnRate
- User.ParticleDistance
```

蓝图实时同步：TransitionRadius

识别网格体距离场，使粒子跟随球体一起缩放
