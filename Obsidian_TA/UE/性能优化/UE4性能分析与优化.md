[[231016--Unreal Engine 4 性能分析与优化.pdf]]

根据这份关于 Unreal Engine 4 (UE4) 性能分析与优化的文档，每一页（或幻灯片）的核心内容整理如下：

### 1. 优化原则与准备

- **第1页（前言）**：强调优化应**尽早且经常进行**，不要等到项目最后；应在目标硬件上测试，并根据检测到的问题进行针对性优化。
- **第2页（准备工作）**：为了让数据更准确，需关闭无用进程、**关闭垂直同步**（`r.vsync 0`）和**帧率平滑**。
- **第3页（准备工作续）**：在编辑器中关闭 Realtime（实时预览），推荐在 **Standalone 模式**下运行游戏，并最小化编辑器以减少干扰。

### 2. 瓶颈定位

- **第4页（GPU瓶颈检测）**：使用 `r.ScreenPercentage 10` 降低分辨率。若游戏帧率突然加快，说明瓶颈在 **GPU**。
- **第5页（Stat FPS）**：使用 `Stat FPS` 查看每秒帧数和刷新一帧的时间（Frame），但这仅作为标准参考，不能用于深入分析。
- **第6页（Stat Unit）**：这是分析核心，将总耗时拆分为 **Game**（逻辑）、**Draw**（CPU渲染）和 **GPU**（GPU渲染）。最接近总耗时的部分即为瓶颈；也可使用 `Stat UnitGraph` 图表查看。
- **第7页（快速分析）**：通过 `Show xxx` 命令（如 `Show StaticMeshes`）切换不同资源的显示，快速判断某类资源是否导致卡顿。

### 3. 性能优化分类处理

- **第8页（因素分类）**：明确不同瓶颈的优化方向——Game看代码蓝图；CPU看 **Draw Call** 和剔除；GPU看材质、灯光和重复绘制（OverDraw）。

#### Draw Call 与 CPU 优化

- **第9-10页（Draw Call原理）**：它是 CPU 向 GPU 发送的渲染命令。Draw Call 过多会导致 CPU 忙于发送命令而过载，这是性能问题的元凶。
- **第11-12页（减少方法）**：核心思想是**批处理**，合并相同材质的模型。需注意动态物体每一帧都要重新合并，往往不适合批处理。
- **第13-15页（UE4批处理工具）**：
    - 使用开发者工具中的 **Merge Actors** 合并模型。
    - 使用 **Instanced Static Meshes**（实例化静态网格体）。
    - 使用 **Foliage**（植被）系统。
- **第16页（查看工具）**：使用控制台命令 `Stat SceneRendering` 查看实时的 Draw Call 数量。

#### 剔除技术（Culling）

- **第17页（定义）**：通过排除不需要渲染的物体来减轻压力。
- **第18-20页（距离剔除）**：类似 LOD 思路，不渲染远距离的小物体。通过 **Cull Distance Volume** 工具，根据物体尺寸（Size，最长边）设定剔除距离。
- **第21-22页（视锥体剔除）**：不渲染屏幕（视野）外的物体。优化方案是精简画面，避免大量物体堆积在同一视野内。
- **第23-24页（遮挡剔除）**：剔除被其他物体挡住的内容。建议用单个大型物体（如一堵大墙）来遮挡后方大量的小物体。
- **第25页（Early Z Pass）**：进一步剔除被挡住的**像素**，减少无效计算。

#### GPU 优化：重复绘制与材质

- **第26-28页（重复绘制 Over Draw）**：**半透材质（Translucent）**不参与深度写入，无法被 Early Z Pass 剔除，会导致严重的重复渲染消耗。建议减少半透面积，大面积贴花推荐用 Plane 模型代替。
- **第29-30页（Over Draw 检查）**：使用 **Shader Complexity** 工具（Alt+8）。颜色由绿向粉白过渡，绿色表示仅绘制一次，粉白色表示严重重复 。
- **第31页（草地优化）**：针对大片透贴草地，近处细化模型减少透贴面积，远处使用简单的 Plane 模型 。
- **第32-33页（材质指令 Instruction）**：关注像素绘制指令数。角色上限一般为 350，环境为 150-200 。
- **第34-36页（材质性能技巧）**：
    - 优先使用 **Static Switch Parameter**，它只计算选中分支；少用 `If` 命令，因为它会同时计算两边路径 。
    - 用纹理贴图替换复杂的数学计算 。
    - 充分利用贴图的 **R/G/B/A 通道**合并四张灰度图，节省内存和指令 。

#### GPU 优化：灯光（Lighting）

- **第37页（灯光属性）**：性能消耗排序为：**静态光（最低） < 固定光 < 动态光（最高）** 。
- **第38-39页（灯光限制）**：尽量限制动态光影响范围。**固定光重叠最多允许 4 次**（占用 RGBA 四个通道），第 5 个重叠的固定光会报红叉警告并转为高消耗的动态处理 。
- **第40-41页（检查工具）**：使用 **Light Complexity** 视图（Alt+7）检查重叠程度，冷色到暖色代表重叠由少变多 。
- **第42页（阴影优化）**：尽量减少动态阴影，关闭地板、贴花等不必要物体的阴影；能使用静态构建就尽量使用 。
